# 调整备忘待办栏展示

## Goal

优化项目详情页备忘 Tab 的左右布局，让左侧待办条目在文本稍长时更容易阅读，同时保留右侧项目备忘的主要编辑空间。

## What I already know

- 用户反馈：备忘 Tab 左侧是待办清单，右侧是项目备忘；待办条目稍微长一点时看起来不方便。
- 现有 `MemoTab.vue` 中待办文本使用单行截断，长内容只能看到一行。
- 现有左右列比例约为 `0.52fr / 1.48fr`，左栏偏窄。

## Assumptions

- 本次只调整项目详情页备忘 Tab 的展示，不改变待办数据结构、拖拽排序、勾选、删除、输入逻辑。
- “最多行数增大到 2”表示待办文本最多显示两行，超过仍截断。
- “宽度适当增大”采用保守比例调整，避免明显压缩右侧项目备忘。

## Requirements

- 待办条目文本最多显示 2 行。
- 超过 2 行的待办文本保持截断，并保留 `title` 悬停查看完整内容。
- 备忘 Tab 左侧待办清单栏宽度适当增加。
- 保持现有交互功能不变：新增、勾选、拖拽排序、删除、空状态、备忘编辑。

## Acceptance Criteria

- [ ] 长待办文本在左侧待办清单中最多显示 2 行。
- [ ] 超过 2 行的文本不会撑开条目或破坏布局。
- [ ] 左侧待办清单栏比当前更宽，右侧项目备忘仍可正常阅读和编辑。
- [ ] `npm run lint` 和 `npm run type-check` 通过。

## Definition of Done

- 前端样式变更集中在备忘 Tab 组件。
- 不引入新的依赖或状态结构。
- 质量检查通过。

## Out of Scope

- 不新增待办详情弹窗、展开/收起交互或富文本能力。
- 不调整待办数据模型或持久化逻辑。
- 不重设计项目备忘编辑器。

## Technical Notes

- 主要影响文件：`src/components/project/MemoTab.vue`。
- 现有文本类包含 `truncate`，需要改为两行截断样式。
- 现有容器使用 CSS grid 自定义列宽，适合小范围调整左栏比例。
