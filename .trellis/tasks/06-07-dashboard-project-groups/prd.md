# 首页项目分组

## Goal

首页项目数量较多时，单一卡片网格会显得拥挤且难以快速定位。为项目增加轻量分组能力，但首页保持连续卡片网格，通过分组筛选和卡片分组标识帮助定位，避免按分组 section 打断网格后产生空白。

## What I Already Know

- 用户认可先实现推荐方案中的“分组”能力。
- 已实现过“按分组 section 展示 + 折叠”的版本，但当某组项目数不是列数或列数倍数时，会在首页产生明显空白。
- 用户确认改为推荐的“分组筛选 + 连续网格”方案。
- 分组筛选初版在暗色模式下选中按钮颜色偏突兀，且筛选条与卡片列表距离略大。
- 当前首页在 `src/components/dashboard/Dashboard.vue` 中渲染可用项目卡片网格，并支持搜索和组内拖拽排序。
- 项目数据模型在 `src/types.ts` 的 `Project` / `ProjectFormValue` 中定义，目前有 `sortOrder` 但没有分组字段。
- 项目持久化、表单草稿、导入导出、排序逻辑集中在 `src/store/useStore.ts`。
- 项目编辑弹窗在 `src/components/project/ProjectFormModal.vue` 中维护基础信息。
- 文案集中在 `src/lib/i18n.ts`。

## Requirements

- 为项目增加可选分组字段，空值统一视为“未分组”。
- 项目创建和编辑表单支持输入/修改分组。
- 首页可用项目保持单个连续卡片网格，不再按分组 section 打断布局。
- 首页在搜索栏附近提供分组筛选控件，至少包含“全部”和已有分组项，每项显示项目数量。
- 选择“全部”时显示所有匹配的可用项目；选择某个分组时只显示该分组中的匹配项目。
- 未填写分组的项目归入“未分组”筛选项。
- 搜索保持全局能力，并与当前分组筛选共同作用。
- 项目卡片上展示轻量分组标识，方便在“全部”视图中识别项目所属分组。
- 现有项目排序继续可用；排序作用于当前可见项目集合，不引入跨分组移动语义。
- 不可用项目仍保留在独立的“需要重新定位的项目”折叠区，不纳入本次分组 UI。
- 旧项目数据、导入数据缺少分组字段时必须兼容，不应导致运行时报错或丢失项目。
- 中英文文案都要补齐。
- 分组筛选控件在暗色模式下应使用协调的选中态颜色，并适当减小与卡片列表的垂直距离。

## Acceptance Criteria

- [ ] 新建/编辑项目时可以填写分组，保存后首页分组筛选中出现对应分组。
- [ ] 未填写分组的项目显示在“未分组”筛选项下。
- [ ] 默认“全部”视图使用连续卡片网格，分组项目数不是列数倍数时不会因分组 section 产生额外空白。
- [ ] 分组筛选项显示项目数量，切换后只展示该分组项目。
- [ ] 搜索项目名称、路径、类型时，会在当前分组筛选范围内过滤，空状态表现正常。
- [ ] 项目卡片在“全部”视图中能看到轻量分组标识。
- [ ] 分组筛选按钮在浅色/暗色模式下都有清晰但不过度突兀的选中态。
- [ ] 分组筛选条与卡片列表之间保持紧凑距离，不造成明显空隙。
- [ ] 开启排序模式后，拖拽排序作用于当前可见项目集合并持久化。
- [ ] 缺少 `group` 字段的旧配置可以正常加载、编辑、导出。
- [ ] `npm run lint` / `npm run type-check` 通过。

## Out Of Scope

- 跨分组拖拽移动项目。
- 单独的分组管理页面、分组重命名/删除批量操作。
- 分组 section 折叠视图。
- 紧凑列表视图。
- 基于运行状态、技术栈或路径自动生成分组。

## Technical Notes

- 优先保持当前 Pinia store 和 Vue 组件风格，不引入新依赖。
- 分组筛选状态作为 Dashboard 局部 UI 状态，暂不需要持久化。
- 分组筛选项顺序可先按项目当前 `sortOrder` 中首次出现的分组顺序推导，保证旧排序语义尽量稳定。
- 当前可见集合拖拽应继续复用现有 `store.reorderProject(projectId, targetProjectId, scopeProjectIds)` 的 scope 参数。
