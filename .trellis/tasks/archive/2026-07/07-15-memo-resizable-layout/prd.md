# 优化备忘布局与拖拽分栏

## Goal

提升项目详情各工具 Tab 的空间利用率：让待办文本获得更多可用宽度，将低频的面板收起控件替换为可直接拖拽的分隔线，并统一备忘、文件、Git、脚本 Tab 的分栏交互。

## Confirmed Facts

- 备忘 Tab 当前使用固定的双栏比例：左侧待办约占 29%，右侧备忘约占 71%。
- 每条待办左右两侧的拖拽和删除按钮在不可见时仍保留固定宽度，待办列表容器和条目还各有横向内边距。
- 当前双栏中间的两个按钮分别收起左、右面板；收起后通过窄栏按钮恢复。
- 文件 Tab 已有基于鼠标移动事件的分栏拖拽实现，但当前不是共享组件或组合式函数。
- 文件 Tab 使用 `event.clientX - 96` 计算栏宽，首次按下会因页面偏移假设与实际位置不一致而跳变。
- Git Tab 仍使用固定左右比例与收起/展开按钮；脚本 Tab 仍使用固定上下比例与收起/展开按钮。
- 项目详情的工具型 Tab 外层错误应用了稳定滚动条槽和右内边距，即使 `overflow-hidden` 也会在右侧预留空间。

## Requirements

- 减少待办条目左右两侧无效占位，在不移除拖拽排序、勾选、双击编辑和删除能力的前提下，为任务文本释放更多横向空间。
- 移除备忘 Tab 的左右面板收起、展开交互。
- 支持拖拽双栏中间分隔线调整左侧待办面板宽度。
- 拖拽过程中保持布局稳定，并限制两侧面板的最小可用宽度。
- 调整后的栏宽仅在当前打开期间保留，不跨插件重开持久化，也不按项目分别存储。
- 文件、Git 使用统一的水平分栏拖拽机制；脚本使用同一交互契约的纵向分栏拖拽机制。
- 工具型 Tab 内容应占满项目详情可用宽度，不为不存在的外层滚动条预留右侧槽位。
- 待办条目不应为默认不可见的删除按钮固定保留右侧文本空间。
- 文件与 Git Tab 的左栏应保持紧凑：文件默认约占 24%、最小 180px；Git 默认约占 28%、最小 200px。

## Acceptance Criteria

- [x] 待办条目在默认状态下不再为不可见操作按钮保留大片横向空白，现有待办操作仍可使用。
- [x] 备忘 Tab 不再显示收起左侧、收起右侧及对应展开按钮。
- [x] 用户可通过鼠标拖拽中间分隔线连续调整左右面板宽度。
- [x] 分隔线具有明确的可拖拽光标与悬停/拖拽反馈，拖拽结束后页面不会继续跟随鼠标变化。
- [x] 调整宽度时左右面板均不会被压缩到不可操作的尺寸。
- [x] 在当前插件打开期间，切换项目或切走再返回备忘 Tab 后仍恢复上次栏宽；重新打开插件后恢复默认比例。
- [x] 现有备忘自动保存、Markdown 编辑、待办增删改、完成状态和拖拽排序行为不回归。
- [x] 项目现有类型检查和构建验证通过。
- [x] 待办文本在删除按钮隐藏时可使用条目右边缘，不再固定保留 `pr-4` 空白；悬停/聚焦时删除仍可操作。
- [x] 文件 Tab 首次按下分隔线不会跳变，并使用与备忘一致的增量计算、边界、清理和会话记忆行为。
- [x] Git Tab 移除左右收起/展开按钮，支持共享水平分隔线拖拽并保持两侧最小可用宽度。
- [x] 脚本 Tab 移除上下收起/展开按钮，支持共享纵向分隔线拖拽并保持脚本列表与终端最小可用高度。
- [x] 备忘、文件、Git、脚本 Tab 在项目详情中占满可用宽度，外层无无效右侧滚动条槽或内边距。
- [x] 四个工具 Tab 在桌面与窄视口下均无分栏跳变、重叠或横向溢出。
- [x] 1024px 桌面视口下，文件与 Git 默认左栏分别接近 24% 和 28%，且首次拖拽仍无跳变。
- [x] 文件与 Git 左栏分别可拖至约 180px 和 200px 下限，同时右侧内容保持可用且容器无溢出。

## Out of Scope

- 不改变待办数据模型、备忘数据模型或持久化格式。

## Notes

- 栏宽记忆决策：使用仅存在于渲染进程内存中的 UI 状态，在项目之间共用当前值，不进入 Pinia 或持久化存储。
- 需求扩展：用户确认将备忘的拖拽机制推广到现有文件、Git、脚本分栏，并消除工具型 Tab 外层右侧空白。
