# 技术设计：备忘可拖拽分栏

## 边界

- `MemoTab.vue` 负责备忘业务内容、双栏结构和待办操作布局。
- 共享分栏组合式函数负责水平与纵向分隔线的通用交互：保存首个面板尺寸、开始拖拽、窗口级移动/结束监听、尺寸约束和卸载清理。
- 组合式函数以稳定的布局键保存渲染进程内存状态，使 `v-if` 销毁并重建 Tab 后仍可恢复；状态不进入 Pinia、项目模型、localStorage 或 preload。
- `MemoTab.vue`、`FilesTab.vue`、`GitTab.vue` 接入水平模式；`ScriptsTab.vue` 接入纵向模式。
- `ProjectDetails.vue` 只在真正滚动的概览内容上应用 `.themed-scrollbar`，工具型 Tab 外层不保留滚动条槽。

## 分栏契约

- 默认保持当前约 29% / 71% 的视觉比例，首次拖拽时从左面板实际渲染宽度开始计算，避免跳变。
- 分隔线使用 Pointer Events，水平模式根据 `clientX`、纵向模式根据 `clientY` 的增量更新首个面板尺寸；容器实时尺寸决定最大值。
- 左栏保留可操作待办列表所需的最小宽度，右栏保留可编辑备忘所需的最小宽度；容器过窄时采用可用范围内的安全值，不产生负宽度。
- 拖拽开始后监听窗口级 `pointermove` / `pointerup` / `pointercancel`，结束或组件卸载时统一清理，避免鼠标移出分隔线后丢失或持续跟随。
- 分隔线暴露 `role="separator"`、水平分栏语义、可拖拽光标，以及 hover / active 视觉反馈。
- 水平与纵向模式分别暴露正确的 `aria-orientation`、光标和方向键；实现共享同一套指针捕获与清理路径。

## 待办行布局

- 勾选框与文本始终参与正常布局。
- 拖拽排序和删除按钮保留原功能，但使用绝对定位覆盖在条目边缘；仅在 hover 或键盘 focus 时显示，不在默认状态占用文本宽度。
- 左侧拖拽入口保留少量必要安全区；右侧删除按钮隐藏时不保留固定文本宽度，显示时覆盖在条目边缘。
- 编辑输入仍占据主要文本区域，Enter、Escape、blur 行为不变。

## 兼容与回滚

- 不修改业务数据和持久化格式，不需要迁移。
- 若共享组合式函数出现问题，可将 `MemoTab` 恢复为原固定 grid 比例；业务数据不受影响。
- 各 Tab 可独立回退到原布局；本任务不改业务数据或终端、Git、文件内容逻辑。

## 验证策略

- 静态验证：`npm run lint`、`npm run build`。
- 浏览器验证：四个工具 Tab 的默认比例、首次按下无跳变、拖拽边界、移出分隔线后结束、切换 Tab 后恢复、待办 hover/focus 操作、编辑与排序回归。
- 视口检查：使用接近截图的桌面尺寸及较窄窗口，确认文本空间、面板最小宽度和无重叠。
- 几何断言：工具 Tab 根节点宽度应等于项目详情内容区宽度，不再因外层稳定滚动条槽损失右侧空间。
