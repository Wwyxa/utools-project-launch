# 完善和优化 Git Tab

## Goal

让用户在不离开 Git Tab 的情况下，准确区分并连续审阅 staged、unstaged 和历史提交的文件 diff，同时保留现有提交、历史、AI 和远程操作能力。

## Background

- 本任务聚焦现有 Git Tab，而不是新增独立 Git 工具。
- 当前 Git Tab 已覆盖：工作区文件状态、单文件与批量暂存/取消暂存/丢弃、diff、提交 staged 变更、AI commit message、提交历史与筛选、提交详情、已有本地分支切换、提交 checkout、remote 管理，以及当前 upstream 的 fetch/pull/push。
- 当前变更文件以单一扁平列表呈现，没有搜索或 staged/unstaged 分组；点击文件会打开 diff 弹层。
- 当前 diff 使用统一文本逐行着色，没有相邻文件导航、行号、并排模式或常驻审阅面板。
- 第一轮实现增加了独立的“工作区审阅 / 提交历史”标签行和提交详情层级；用户反馈前者浪费垂直空间，后者仍使历史提交审阅路径过深。
- 现有单提交 AI 会读取该提交的文件列表和历史文件 diff；提交树选择分析主要使用所选提交元数据，却附加当前工作区文件/diff，两者语义不一致。

## Requirements

### R1. 变更文件组织与定位

- 工作区文件列表按“已暂存 / 更改”分组；同一文件同时存在 staged 与 unstaged 变更时可在两组分别出现。
- “已暂存”保持在“更改”上方，但 staged 文件为零时不渲染“已暂存”折叠栏。
- 变更列表支持按相对路径搜索、独立折叠 staged/unstaged 分组，并可通过键盘切换上一个/下一个可见文件。
- 搜索框的图标、输入文本和 placeholder 必须有稳定间距，不得重叠。
- 变更列表顶部不显示“上一个/下一个可见文件”按钮；文件导航保留在审阅区域或键盘路径中。
- 搜索与折叠只影响呈现和导航，不改变批量暂存、取消暂存或丢弃操作所针对的完整实时 Git 状态。

### R2. 范围准确的工作区审阅

- 工作区 diff 必须按所选 staged 或 unstaged 范围读取和展示，不得将两种范围合并后作为提交预览。
- 点击左侧文件后，右侧自动进入“工作区审阅”并显示该文件、该范围的 diff。
- 暂存当前文件后，选择跟随同一文件迁移到 staged 范围；取消暂存后迁移到 unstaged 范围。
- 丢弃当前文件后，选择同组下一个可见文件；没有后继项时显示工作区审阅空状态。

### R3. 共享增强 Diff 查看器

- 工作区和历史提交共用增强 unified diff 查看器，提供旧/新行号、hunk 分隔与跳转、长行换行开关，以及明确的加载、空内容和不可预览状态。

### R4. 稳定的右侧审阅导航

- Git Tab 当前空间已被左右主区域占满；优化不得新增第三栏或持续挤压现有列表与历史区域。
- 不在提交树原菜单栏上方增加独立标签行；在原菜单栏左侧放置“提交树 / 审阅”紧凑模式切换，提交树在首位且为默认模式。
- 原菜单栏右侧按钮随模式切换：提交树模式显示提交选择、筛选、AI 和滚动操作；审阅模式显示与当前 diff 相关的文件导航及查看操作。
- 右侧同一时刻只显示提交树或审阅内容，不拆出额外并排窗口。
- 工作区审阅与提交历史分别保留选择和滚动位置，切换标签时不重置另一上下文。
- 每次进入 Git Tab 时右侧默认显示提交历史，不根据工作区是否有变更自动切换；用户点击左侧文件后才进入工作区审阅。

### R5. 历史提交审阅

- 移除独立提交详情页。单击提交记录直接在该记录下方展开/收起变更文件列表，并保持提交树上下文可见。
- 点击展开列表中的文件进入共享审阅区域查看该提交文件 diff；返回提交树时恢复展开提交和滚动位置。
- 提交记录 tooltip 提供比列表更完整的作者、绝对时间、refs 和 commit message/body 信息；不重复显示列表已有的 hash。
- tooltip 使用可进入的 hover card：延迟出现，允许鼠标从提交行移入浮层并选择/复制内容；只有行与浮层均失去悬浮后才延迟关闭，`Escape` 可关闭，不使用点击固定。
- checkout 仅从提交记录的右键菜单触发，不增加常驻或悬浮操作按钮；提交行前的勾选框只用于 AI 多选，不触发 checkout。
- checkout 菜单沿用现有 clean/dirty worktree 校验和危险操作确认，不新增其他历史写操作。

### R6. AI 分析收敛

- 移除提交详情页内的单提交 AI，总结入口统一为提交树现有的提交选择分析。
- 未选择提交时，AI 分析当前工作区的文件和 diff；生成弹窗中简短提示“未选择提交，将分析当前工作区变更”。
- 手动选择一条或多条提交时，分析上下文应来自所选提交的元数据、文件列表和有界历史 diff，不得附加无关的当前工作区 diff；弹窗简短提示当前分析所选提交及数量。
- 选择多条提交并包含 diff 时，所有所选提交的 commit message 必须优先完整进入上下文，diff 只能使用剩余预算并可被截断。
- diff 超出上下文预算时，弹窗显示简短提示“Diff 已截断，所有提交信息已保留”，不得静默截断或丢弃任一所选提交的 commit message。
- 提交筛选继续用于缩小可见范围，并保留“选择全部可见提交”作为批量选择入口。

### R7. 兼容与安全

- 保持现有项目数据和 Git 仓库操作兼容，不破坏已有功能。
- 本轮不新增 Git 写操作种类；现有丢弃与强制切换等危险操作继续使用应用内确认和既有错误恢复约定。
- 现有 commit message AI、提交筛选与选择、remote 管理及 fetch/pull/push 行为必须保留；checkout 只迁移入口，不改变底层语义。

## Acceptance Criteria

- [ ] AC1: staged/unstaged 分组显示正确；同一文件两种范围都有变更时在两组各有一项，计数和整文件操作状态与 Git 状态一致。（R1）
- [ ] AC2: staged 为空时不显示“已暂存”折叠栏；搜索图标与提示文本不重叠；变更列表顶部没有上/下文件按钮；清除搜索后恢复完整列表，批量操作仍覆盖完整实时 Git 状态。（R1）
- [ ] AC3: 选择 staged 项只显示 `git diff --cached -- <path>` 范围，选择 unstaged 项只显示工作区及未跟踪文件范围，两者不会合并。（R2）
- [ ] AC4: 暂存/取消暂存后选择跟随同一文件切换范围；丢弃后选择同组下一个可见项或显示空状态，不显示过期 diff。（R2）
- [ ] AC5: 工作区和历史提交使用同一 enhanced unified diff 查看器，正确显示旧/新行号，支持 hunk 前后跳转和长行换行切换，并覆盖加载、无 diff、二进制/不可预览和失败状态。（R3）
- [ ] AC6: 右侧不增加额外标签行；原菜单栏左侧依次显示“提交树 / 审阅”，默认提交树，右侧动作随模式正确切换，并保留两个模式各自的选择和滚动位置。（R4）
- [ ] AC7: 单击提交记录在原位展开/收起变更文件列表，不进入详情页；历史文件 diff 返回后恢复展开项、提交树滚动位置和筛选/选择状态。（R5）
- [ ] AC8: 提交 tooltip 不重复 hash，内容完整、可移入和交互；提交右键菜单可执行 checkout，并保持 dirty-worktree 确认与错误反馈。（R5、R7）
- [ ] AC9: 提交树 AI 未选择提交时分析当前工作区并显示简短范围提示；选择后只分析所选提交，全部 commit message 优先保留，diff 超限时截断并显示简短提示；详情页单提交 AI 不再存在。（R6）
- [ ] AC10: 窄窗口下仍保持两栏结构，工具栏模式切换、动作区、文件路径、diff gutter、展开列表和 tooltip 不重叠或产生不可达控件。（R4、R5）
- [ ] AC11: 现有整文件暂存/取消暂存/丢弃、提交、commit message AI、提交筛选与选择、remote 和同步操作继续通过原有 store/bridge 边界工作，危险操作确认不回退。（R7）

## Out of Scope

- 创建、重命名或删除分支；stash；merge/rebase 与冲突解决；tag；cherry-pick；revert/reset；blame；worktree。
- 按 hunk 或行部分暂存/取消暂存。
- 并排 diff、代码编辑或评论能力。
- 新增 force push、rebase pull、push-upstream 等远程写操作。
