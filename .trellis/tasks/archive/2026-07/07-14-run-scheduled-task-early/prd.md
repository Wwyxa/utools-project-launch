# 为计划任务添加提前执行

## Goal

允许用户对已经生成、尚未到执行时间的计划条目提前执行，并让该次执行沿用原计划记录，避免到点重复运行。

## Confirmed Facts

- 自动化任务会为当天生成包含 `plannedAt` 和状态的计划条目，并在任务卡片上逐条展示。
- 首页任务概览的“下次执行”列表直接持有每个任务最早的未来待执行条目，包括原始 `entryId`。
- 现有“立即执行”属于任务级手动运行，会额外创建一条 `manual` 计划记录，不会消费已经生成的待执行条目。
- 已有执行核心可以按 `entryId` 运行待执行条目；条目进入 `running` 后不会再被定时调度，到达终态后会写入执行历史。
- 同一项目只允许一个自动化任务运行，运行冲突由现有项目级活动运行状态拦截。

## Requirements

- 对当天所有未来的待执行计划条目分别提供“提前执行”入口，由用户明确选择目标条目。
- 首页任务概览的“下次执行”列表也必须为每个列出的计划条目提供“提前执行”入口。
- 提前执行必须复用被选择的计划条目，不新增手动计划条目。
- 提前执行后的状态、历史、通知、脚本顺序、自动输入、关键词退出和超时行为与正常到点执行保持一致。
- 提前执行开始后，原计划时间到达时不得再次执行同一条目。
- 不允许对已运行或已结束的条目重复提前执行。
- 当同一项目已有自动化任务运行时，不得开始提前执行，并应向用户提供明确反馈。
- 关闭定时执行不影响用户显式提前执行已生成条目，与现有任务级“立即执行”保持一致。
- 提前执行不增加二次确认步骤。

## Acceptance Criteria

- [x] 用户可从当天任意一个未来待执行计划条目触发提前执行。
- [x] 用户可从首页任务概览的“下次执行”列表直接提前执行对应计划条目，无需先进入项目任务页。
- [x] 触发成功后，被选条目进入运行状态，且不会生成额外的 `manual` 条目。
- [x] 执行完成或失败后，被选条目和历史记录反映正确结果，同时保留原 `plannedAt`。
- [x] 到达原计划时间时，该条目不会再次执行。
- [x] 非待执行条目、无可执行脚本或同项目已有任务运行时，入口不可用或操作被可靠拒绝。
- [x] 中英文界面均有准确的操作名称、提示和无障碍标签。
- [x] 首页“下次执行”列表仍可通过整行点击或键盘操作进入对应项目任务页，提前执行按钮不会触发该导航。
- [x] 现有任务级“立即执行”行为保持不变。

## Out Of Scope

- 修改计划生成算法或计划时间。
- 为历史条目增加重新执行能力。
- 改变同项目只允许一个自动化任务运行的并发规则。
