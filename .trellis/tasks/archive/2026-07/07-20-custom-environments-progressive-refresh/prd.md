# 自定义开发环境与增量刷新

## Goal

让用户能够在现有内置开发工具之外配置自己的环境检测项，并让环境刷新结果按单项完成顺序即时展示，减少等待整批检测结束时的不确定感。

## Background

- 当前环境页支持 Node.js、npm、pnpm、Yarn、Python、pip、Go、Git 和 Docker 等固定工具。
- 环境检测由 bridge 调用命令完成；Store 等待整批检测 Promise 返回后一次性替换 `environmentResults`，因此页面虽然显示检测过程，却只能在全部检测结束后统一显示结果。
- preload 内部已使用 `Promise.all` 并发检测各环境，但 bridge 合约只提供整批结果，没有单项完成通知。
- 每个内置工具先执行版本命令，再通过 `where`（Windows）或 `which`（其他平台）解析路径；每条命令当前有 5 秒超时。
- 当前刷新状态只有全局 `environmentRefreshing`，页面不能区分各环境是等待中、检测中还是已完成。
- 环境偏好通过 `utools-project-launch.environment-settings.v1` 持久化，目前只保存启用的固定工具 key。

## Requirements

- 支持用户新增、编辑和删除自定义环境检测项。
- 自定义项配置字段为显示名称、检测命令和版本参数；系统使用 `where`（Windows）或 `which`（其他平台）自动解析可执行路径，不开放完整 shell 命令。Windows 中解析到 `.cmd` / `.bat` CLI shim 时允许通过受限 `cmd.exe` 启动，但必须拒绝 cmd 元字符和变量展开。
- 自定义项使用独立稳定 ID；显示名称、可执行文件必须为非空值，版本参数按参数 token 保存，命令和参数不得包含 shell 管道、重定向、串联或控制字符。
- 自定义环境检测项是当前开发机的全局配置，所有项目共享，不支持项目级覆盖或扩展。
- 设置页是自定义环境的唯一管理入口，负责新增、编辑、删除和启用状态；环境检测页只负责展示结果与触发刷新。
- 设置与检测界面保持紧凑、高信息密度，不使用占据大片空白的独立区块、超高表单或宽松卡片布局。
- 设置页将默认环境与自定义环境放在同一个紧凑网格中，使用一致的卡片结构；仅自定义项显示尽量小的“自定义”标志，默认项平时不显示来源标志，存在命令覆盖时显示紧凑的“已修改”。不再在下方建立独立的“自定义环境”区域。
- “新增环境”按钮放在环境设置区右上角；点击环境卡片主体直接打开该项弹窗，卡片内启用复选框独立操作且不触发弹窗。
- 统一卡片表面不放编辑或删除图标；自定义环境的删除入口位于编辑弹窗内，并继续要求二次确认。
- 默认环境弹窗中名称和内置 key 固定，只允许覆盖可执行文件与版本参数；提供“恢复默认”操作。存在覆盖时卡片显示“已修改”，清除覆盖后恢复 preload 内置平台命令。
- 默认环境覆盖属于用户输入，使用与自定义环境相同的受限校验边界；原生程序 direct spawn，Windows `.cmd` / `.bat` 仅走受限 shim 启动，不进入受信任内置命令的任意 shell 执行路径。
- 新增、编辑或重新启用环境项后，使该项旧结果失效；下次进入环境页时仅自动检测这些缺少有效结果的启用项，不在设置页保存时后台执行命令。
- 自定义环境检测项可持久化，并能像内置环境一样启用、禁用和展示检测结果。
- 刷新全部环境时，各检测项独立执行；任一检测完成后立即更新该项，不等待其他检测完成。
- 刷新期间为每个检测项展示准确的独立状态，同时保留全局刷新按钮的整体忙碌状态。
- 手动刷新已有结果时保留上一次版本、路径和检测时间，并将该项标记为检测中；该项完成后立即替换为新结果。首次检测没有旧结果时，仅未完成的卡片显示骨架。
- 单项检测失败或超时不得阻塞其他环境结果展示。
- 内置和自定义环境的每条检测命令统一使用现有 5 秒超时，不提供单项或全局超时设置。
- 环境检测需要限制同时运行的本机进程数量；自定义项数量不设置产品级上限，超出并发容量的检测项排队执行。
- 保持现有内置环境配置和已保存启用项兼容。

## Acceptance Criteria

- [ ] 用户可通过显示名称、可执行文件和版本参数配置至少一个自定义环境检测项，重新打开插件后配置仍存在。
- [ ] 无效或包含 shell 操作符的自定义命令配置无法保存，并显示明确的字段错误。
- [ ] 自定义项不需要用户填写路径检测命令，检测成功后可自动展示可执行路径。
- [ ] Windows 中终端可见的 `code`、`npm` 等 `.cmd` / `.bat` CLI shim 可被检测，且 cmd 元字符或变量展开参数在执行前被拒绝。
- [ ] 自定义环境可被启用、禁用、编辑和删除，并参与环境页检测与结果展示。
- [ ] 默认和自定义环境在设置页使用同一网格与卡片样式，来源标志清晰；自定义环境不再重复显示在独立分区中。
- [ ] 环境设置区右上角可直接新增环境；点击任一环境卡片可打开对应弹窗，点击复选框只切换启用状态。
- [ ] 自定义环境编辑弹窗内可发起删除并完成二次确认；统一网格卡片上不显示编辑或删除图标。
- [ ] 默认环境可覆盖检测命令和版本参数，名称不可修改；覆盖配置可持久化并一键恢复默认，覆盖检测不进入一般 shell，仅允许经过元字符拒绝的 Windows CLI shim。
- [ ] 常规和窄窗口下，自定义环境管理区与检测卡片保持紧凑，无大片无效留白、文本重叠或横向溢出。
- [ ] 新增、编辑或重新启用环境后进入环境页，受影响项自动开始检测，未变化且已有结果的环境不重复检测。
- [ ] 点击刷新后，先完成的环境项先显示最终状态，其余项继续显示检测中。
- [ ] 手动刷新期间已有结果保持可见且布局不跳变；首次检测时每张卡片在自身完成后立即从骨架切换为结果。
- [ ] 某个环境命令失败或超时时，其他环境仍能独立完成并及时展示。
- [ ] 刷新全部结束后，全局刷新状态正确结束，每个启用项都有明确结果。
- [ ] 原有内置环境与旧版环境偏好数据升级后仍可正常使用。

## Out of Scope

- 项目级自定义环境检测、全局项继承和项目覆盖。
- 管道、重定向、命令串联、初始化脚本等完整 shell 表达式。
- 自定义项拖拽排序、导入导出和预设市场。
