# 暗色界面视觉层级设计

## Boundary

改动集中在 `src/index.css` 的 `.dark` 语义令牌与仪表盘暗色专用覆盖。Vue 组件继续消费现有 Tailwind 语义类，不修改模板结构、状态或事件。

## Palette Strategy

- 以 GitHub 暗色模式的中性石墨灰层级为参考，提高最低表面和页面背景的明度。
- 保持从页面背景、低层容器、标准表面到高层/悬停表面的单调递增关系。
- 边框使用比相邻表面略亮的中性灰，避免发白描边。
- 正文保持高对比，次级文字适度提亮；绿色主色令牌继续沿用当前色相。
- 工具栏、筛选标签、滚动条和输入控件复用同一表面阶梯，消除局部近黑块。
- 隐藏操作条和浮动菜单使用最高层表面配合边框与阴影；按钮默认图标使用次级文字色，hover/focus 再切换为正文、主色或危险色。

## Compatibility

- 不修改令牌名称，因此现有 Tailwind 类和组件 API 保持兼容。
- 不修改亮色 `@theme` 值。
- 不修改代码预览和 syntax 令牌，避免影响 diff 与代码阅读体验。
- 不改变隐藏按钮的显隐逻辑、定位或可访问性属性，仅验证其颜色令牌在新暗色阶梯中的表现。

## Trade-offs

- 全局令牌调整会影响所有暗色页面，这是实现全局一致性的预期结果；人工检查需覆盖仪表盘和项目详情两个主要入口。
- 不新增第二套 GitHub 主题或用户可调亮度，避免配置和维护成本。

## Rollback

改动不涉及数据或状态迁移，可直接恢复 `.dark` 令牌与仪表盘覆盖值。
