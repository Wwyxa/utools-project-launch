# 优化暗色界面视觉层级

## Goal

提升应用暗色模式的明度、层级辨识度和整体观感，参考 GitHub 暗色模式的中性深灰基底，同时保留当前紧凑布局、绿色强调色和组件视觉语言。

## Background

- 当前暗色表面从 `#0d0e0f` 到 `#2a2d2e`，背景与低层容器整体偏黑，页面大面积区域显得暗沉。
- 应用已通过 `src/index.css` 中的语义颜色令牌驱动主要界面；项目详情、设置、环境、终端和仪表盘组件普遍复用这些令牌。
- 仪表盘筛选按钮另有少量 `.dark` 硬编码颜色，需要与新表面阶梯同步调整。
- 项目卡片、Git 文件行和备忘录条目包含默认隐藏、在 hover 或 focus 时显现的快捷操作按钮；脚本菜单及各类上下文菜单也使用高层浮动表面。
- 用户提供的 GitHub 暗色界面仅作为亮度、对比度和层级参考，不要求复制 GitHub 的布局或品牌样式。

## Requirements

- R1：仅优化暗色模式，不改变亮色模式配色。
- R2：提高页面背景、卡片、工具栏、输入框、弹层之间的明度区分，避免大面积近黑色连成一片。
- R3：采用接近 GitHub 暗色模式的中性偏冷石墨灰方向，继续使用现有绿色作为主强调色和运行状态色。
- R4：保留现有布局、圆角、间距、字体、阴影尺度、组件结构和交互行为。
- R5：优先调整现有语义令牌；仅对无法随令牌统一变化的暗色硬编码样式做局部修正。
- R6：暗色下正文、次级文字、边框、禁用态和悬停态仍需清晰可辨，不用过亮边框制造割裂感。
- R7：代码预览与语法高亮继续使用现有 GitHub 风格令牌，不在本次重新设计范围内。
- R8：适配默认隐藏的悬停/聚焦操作按钮及浮动菜单，显现后需与卡片拉开层级，普通、危险、禁用图标均清晰可辨。

## Acceptance Criteria

- [ ] AC1：仪表盘背景不再接近纯黑，工具栏、项目卡片和页面背景形成清楚但克制的三层关系。
- [ ] AC2：项目详情、设置、环境、终端、弹窗和浮层能够通过共享令牌同步获得新的暗色层级。
- [ ] AC3：筛选标签、输入框、图标按钮及其 hover/active 状态与新暗色表面协调，边界清晰。
- [ ] AC4：绿色主色和运行状态仍是界面的主要视觉强调，不引入新的主色体系。
- [ ] AC5：亮色模式视觉不发生变化，功能、布局和响应式行为不发生变化。
- [ ] AC6：`npm run lint` 与 `npm run build` 通过。
- [ ] AC7：在桌面与紧凑窗口下人工检查仪表盘和项目详情，确认无文字重叠、裁切或层级丢失。
- [ ] AC8：逐项触发项目卡片快捷操作条、Git 文件行操作组、备忘录行内按钮、脚本下拉菜单和上下文菜单，确认背景、边框、阴影、图标及 hover/focus 状态清楚且不突兀。

## Out of Scope

- 调整页面布局、组件密度、导航结构或功能流程。
- 重做品牌色、图标、字体或代码语法配色。
- 新增主题切换选项、依赖或独立主题配置系统。
