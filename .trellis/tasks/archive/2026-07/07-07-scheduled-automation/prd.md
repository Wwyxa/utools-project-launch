# 自动任务功能

## Goal

为项目增加自动任务能力，让用户可以为某个项目配置每天定时或随机时间、按指定次数执行一个或多个脚本，并在脚本需要交互输入时按预设内容自动完成输入。

用户价值：把重复的项目维护、签到、同步、构建、测试或巡检脚本从手动点击变成可审计、可暂停、可复用的自动执行流程。

## Confirmed Facts

- 项目数据已经包含 `scripts`、`env`、`path`、脚本 `cwd` 和脚本运行状态。
- 脚本运行通过 `store.launchScript(projectId, scriptId)` 调用 `projectBridge.runCommand(...)`，preload 使用 `child_process.spawn(..., { shell: true })` 启动进程。
- 运行日志通过 `started`、`stdout`、`stderr`、`stdin`、`exit`、`error` 事件回传到 store，并显示在内置终端面板。
- 交互输入已经存在：`store.sendScriptInput(projectId, scriptId, input)` 会调用 `projectBridge.sendProcessInput(pid, input)`，preload 会向子进程 stdin 写入一行并记录 `stdin` 日志。
- 当前项目持久化有兼容性校验脚本，新增配置字段需要兼容旧项目数据和导入导出。
- 当前没有独立后台服务迹象；插件退出时会请求停止运行中的脚本。
- 项目详情页当前有 `info`、`scripts`、`files`、`git`、`memo` 标签页；自动任务将作为新的项目详情标签页承载精细配置和历史。
- 首页顶部工具栏当前已有搜索、排序、环境、设置、刷新、新增等入口；自动任务概览按钮可以放在同一区域，提供跨项目入口。

## Requirements

- 用户可以为某个项目创建、编辑、启用、禁用、删除自动任务。
- 每个自动任务可以选择一个或多个现有项目脚本作为执行目标；同一项目内多个脚本必须串行执行，不同项目的自动任务可以并行执行。
- 每个自动任务支持每天固定时间执行；固定时间模式使用起始时间、每日次数和固定间隔生成当天计划。
- 每个自动任务支持每天在用户配置的时间窗口内随机选择执行时间；随机模式支持配置不同次数之间的最小、最大执行间隔。
- 每个自动任务支持配置每天执行次数；多次执行时要能避免同一脚本并发重复启动，并确保生成的计划满足配置的间隔约束。
- 当每日次数、时间窗口、最小/最大间隔等约束无法同时满足时，配置表单必须在保存前阻止保存，并提示用户调整配置。
- 如果插件未运行导致计划时间错过，MVP 不自动补跑，只记录错过原因，并等待下一次正常计划。
- 同一项目内串行执行多个脚本时，如果某个脚本失败，自动任务必须停止后续脚本并记录失败原因。
- 如果同一项目已有自动任务仍在执行，后续到点的同项目计划必须跳过本次执行并记录原因，不排队等待。
- 每个自动任务支持配置单脚本最大运行时长，默认 30 分钟；脚本超时后应停止该脚本、记录失败，并停止同项目后续脚本。
- 每个自动任务支持为需要输入的脚本配置预设输入序列；输入序列按脚本级配置，并在对应脚本运行期间自动发送到该脚本进程。
- 输出匹配式自动输入必须配置超时时间；匹配超时后该脚本判定失败，并按串行失败策略停止后续脚本。
- 每个脚本的自动输入步骤全部完成后，支持继续等待输出关键词并自动退出/停止该脚本。
- 关键词退出不单独配置超时时间；如果一直等不到退出关键词，由单脚本最大运行时长兜底处理。
- 自动输入明文保存和显示；发送过的输入应出现在现有终端日志或自动任务运行记录中。
- 用户可以查看自动任务的启用状态、下一次计划时间、最近执行结果、失败原因和最近 20 条轻量执行历史。
- 自动任务执行完成、失败或跳过时应支持弹出系统层面通知；优先复用现有 `window.utools.showNotification` 能力。
- 每个自动任务应有独立通知开关，默认开启；关闭后该任务仍记录历史，但不弹系统通知。
- 项目详情页应新增“自动任务”标签页，作为单个项目自动任务的精细管理入口。
- 首页顶部应新增自动任务按钮，用于查看所有项目自动任务的概览；概览不需要完整编辑能力，但应支持点击进入对应项目的自动任务详情。
- 自动任务执行应复用现有脚本运行、停止、日志和环境变量机制，避免另起一套命令执行路径。
- 自动任务配置必须随项目持久化，并兼容旧项目配置、导入和导出。
- 当项目路径不可用、脚本不存在、脚本命令为空、脚本正在运行或输入发送失败时，自动任务应记录失败或跳过原因，不能静默失败。

## Acceptance Criteria

- [ ] 可以在某个项目下配置至少一个自动任务，选择单个或多个脚本。
- [ ] 固定时间模式可以保存起始时间、每日次数和固定间隔，生成可展示的当天计划，展示下一次执行时间，并在当天到点后触发脚本。
- [ ] 随机时间模式可以保存时间窗口、每日次数、最小执行间隔、最大执行间隔，并为当天生成满足约束且可展示的固定随机计划。
- [ ] 当计划约束不可满足时，用户无法保存自动任务，并能看到需要调整时间窗口、次数或间隔的提示。
- [ ] 插件启动时发现已错过的计划不会自动补跑，会记录错过原因并继续计算下一次计划。
- [ ] 自动任务触发后，脚本使用现有 `launchScript` 路径启动，日志进入现有终端日志体系；同一项目内多个脚本按配置顺序串行执行。
- [ ] 串行执行中任一脚本失败时，后续脚本不会继续执行，自动任务历史记录失败脚本和原因。
- [ ] 同一项目已有自动任务执行中时，新到点的同项目计划会跳过并记录原因，不进入等待队列。
- [ ] 单个脚本超过配置的最大运行时长时，会被停止并记录超时失败；默认最大运行时长为 30 分钟。
- [ ] 对需要输入的脚本，可以为每个脚本单独配置预设输入序列；预设输入会按配置发送到运行进程，并能在日志中看到明文发送记录。
- [ ] 输出匹配式输入在超时未匹配到目标文本时，会记录超时失败，并停止同项目后续脚本。
- [ ] 脚本完成全部自动输入步骤后，可以按配置等待输出关键词并自动停止脚本；关键词退出不会早于输入步骤完成触发。
- [ ] 关键词退出等待不设置独立超时；一直未匹配时由单脚本最大运行时长触发超时失败。
- [ ] 同一脚本正在 `RUNNING` 或 `STOPPING` 时，自动任务不会重复启动该脚本，并会记录跳过原因。
- [ ] 插件重启后，已保存的自动任务配置仍能加载；旧项目数据不含自动任务字段时仍能正常加载。
- [ ] 项目导出导入保留自动任务配置，兼容性校验覆盖新增字段。
- [ ] 用户可以临时禁用自动任务，禁用后不会继续触发。
- [ ] 自动任务保留最近 20 条轻量执行历史，包含任务名、计划时间、实际开始/结束时间、脚本结果、跳过或失败原因。
- [ ] 自动任务完成、失败或跳过时可以弹出系统通知；通知不可用时不影响任务执行，仍写入历史。
- [ ] 每个自动任务可以单独开启或关闭系统通知，新建任务默认开启通知。
- [ ] 项目详情页有“自动任务”标签页，可以管理该项目的自动任务配置和历史。
- [ ] 首页顶部有自动任务概览按钮，可以展示跨项目自动任务摘要，并能跳转到对应项目的自动任务标签页。

## Out of Scope

- 跨系统重启或插件完全未运行时的后台调度保证，除非后续明确需要 OS 级计划任务。
- 自动识别命令提示词并智能决定输入内容；MVP 只使用用户预先配置的输入序列。
- 复杂工作流编排，如条件分支、依赖 DAG、失败重试策略矩阵。
- 云同步、多人审批、远程执行。

## Decisions

- 自动输入触发策略已决定：同时支持固定延迟/间隔发送和输出匹配后发送，MVP 默认使用固定延迟/间隔；输出匹配作为更精细的交互模式。
- 调度可靠性边界已决定：MVP 仅在 uTools 插件运行期间触发，不保证插件关闭或系统重启后仍能执行。
- 多脚本执行顺序已决定：同一项目内多个脚本串行执行；不同项目的自动任务可以并行执行。
- 执行结果保留范围已决定：需要独立轻量历史，MVP 保留最近 20 条执行记录。
- 串行失败策略已决定：同一项目内任一脚本失败时停止后续脚本。
- 系统通知需求已提出：自动任务完成、失败或跳过时支持系统层面通知，初步优先复用 `window.utools.showNotification`。
- 错过计划处理已决定：插件未运行导致错过计划时间时不补跑，只记录错过。
- 通知控制已决定：每个自动任务有独立通知开关，默认开启。
- 自动任务入口已决定：项目详情页新增“自动任务”标签页作为精细管理入口；首页顶部新增自动任务概览按钮作为跨项目入口。
- 随机计划生成已决定：每天生成固定随机计划，用户可看到当天 N 个时间点；随机计划必须满足最小、最大执行间隔约束。
- 固定时间多次执行已决定：使用起始时间、每日次数和固定间隔生成当天计划。
- 自动输入配置粒度已决定：按脚本级配置输入序列。
- 自动输入显示策略已决定：预设输入明文保存和显示，不做隐藏或脱敏。
- 不可满足约束处理已决定：保存前校验并阻止保存，提示用户调整时间窗口、次数或间隔。
- 同项目运行冲突处理已决定：同一项目已有自动任务执行中时，后续到点计划跳过并记录原因，不排队。
- 输出匹配输入超时处理已决定：匹配超时则脚本失败，并停止同项目后续脚本。
- 单脚本最大运行时长已决定：默认 30 分钟，可配置；超时停止脚本并记录失败。
- 关键词退出需求已提出：脚本级自动输入全部完成后，支持根据输出关键词自动退出/停止脚本。
- 关键词退出超时策略已决定：不单独配置退出超时，由单脚本最大运行时长兜底。

## Open Questions

- 暂无阻塞规划的问题；进入实现前需要用户审阅 PRD、design 和 implement。

## Notes

- 初步设计倾向：把自动任务作为 `Project` 的新增配置字段，调度器在前端/store 层基于已保存配置计算下一次触发，并通过现有 `launchScript` / `sendScriptInput` 执行。
- 风险点：MVP 不引入 OS 级计划任务或常驻进程；用户需要保持插件运行，自动任务才会触发。
- UI 入口：在项目详情页新增“自动任务”标签页，集中展示任务列表、启用状态、下一次计划、历史和编辑入口；首页顶部新增概览按钮，展示所有项目自动任务摘要并支持跳转。
