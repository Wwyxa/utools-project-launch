# 首页待办快捷入口

## Goal

在首页增加一个待办快捷入口，让用户不用逐个打开项目详情，也能快速看到所有项目尚未完成的待办，并可直接将待办勾选完成或跳转到对应项目继续处理。

## Confirmed Facts

- 项目数据模型已有 `todos?: TodoItem[]`，`TodoItem` 包含 `id`、`text`、`completed`。
- `store.todos` 按项目 ID 存储待办，`loadProjects()` 会从项目配置同步 `todos`，待办编辑动作会持久化项目配置。
- 项目详情概览已显示待办未完成数 / 总数，备忘页 `MemoTab` 已支持添加、勾选、编辑、删除、拖拽排序待办。
- 首页 `Dashboard.vue` 顶部工具栏已有自动化总览按钮和 Teleport 弹层，可作为待办总览入口的交互参考。
- 应保留现有项目搜索、分组筛选、排序、自动化总览、环境、设置、刷新、创建项目等入口的可用性。

## Requirements

- 首页工具栏新增待办快捷按钮，按钮展示全部可见项目中的未完成待办数量。
- 点击按钮后展示一个首页级待办总览，按项目聚合列出未完成待办。
- 待办总览应只统计当前设备可见项目，且以未完成待办为主，不展示已完成待办。
- 每条待办应展示所属项目名称和待办文本，文本过长时保持可读且不挤压布局。
- 首页总览待办条目和项目分组应保持紧凑列表样式，不为每条待办或每个项目使用卡片式边框、重复操作提示或大间距。
- 用户应能在首页总览中直接勾选未完成待办；勾选后该待办完成状态应通过现有待办持久化逻辑保存，并从未完成列表中移除。
- 用户应能从总览快速打开对应项目的待办管理位置，继续使用现有项目备忘页处理待办。
- 当没有未完成待办时，总览应展示空状态，而不是空白面板。
- 文案需要覆盖中文与英文 locale。
- 首页在窄屏和常规桌面宽度下不应出现工具栏文字挤压或弹层内容溢出。

## Acceptance Criteria

- [x] 首页顶部出现待办快捷按钮，按钮计数等于所有可见项目中 `completed === false` 的待办数量。
- [x] 点击待办快捷按钮可打开/关闭待办总览弹层，Escape 或关闭按钮可关闭弹层。
- [x] 总览按项目分组展示未完成待办；没有未完成待办的项目不出现在列表中。
- [x] 总览待办条目和项目分组采用紧凑列表样式，不显示重复的“打开项目待办”提示，不使用项目卡片式间隔。
- [x] 在总览中勾选待办后，该待办被标记为完成、全局未完成计数减少，并不再显示在未完成列表中。
- [x] 点击项目或待办入口能进入对应项目的待办管理位置。
- [x] 没有未完成待办时显示本地化空状态。
- [x] 中文和英文文案均完整，未出现未翻译 key。
- [x] `npm run build` 通过。

## Notes

- 建议保持轻量任务，PRD-only 即可。
- 推荐实现方式：复用首页自动化总览的工具栏按钮和弹层模式；复用 `store.toggleTodo` 完成状态切换；新增 store 方法或扩展现有方法，让首页可跳转到项目详情的备忘 / 待办页。
