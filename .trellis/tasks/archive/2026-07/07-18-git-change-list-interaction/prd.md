# Improve git change list interaction

## Goal

让 Git 提交树中的变更文件列表支持同时展开多个提交，并提供平铺列表与可折叠目录层级的树形列表两种查看方式，避免展开状态切换导致图形和内容错位。

## Confirmed Facts

- 提交树和其内联变更列表由 `src/components/project/GitTab.vue` 渲染。
- 当前实现使用单个 `selectedCommitHash`、`selectedCommitFiles`、加载状态和错误状态保存展开内容；点击另一条提交会替换这些状态，从而收起原先的列表。
- 提交树 SVG 的行坐标目前只为唯一的展开提交增加高度偏移，因此多项展开需要按每个展开块的实际高度累计偏移。
- 点击内联文件会打开对应提交的 diff；多项展开后，文件操作必须仍携带所属提交 hash。
- 当前树形模式会完整递归渲染所有虚拟目录，目录项不可交互，且内联列表高度上限为 `144px`。

## Requirements

1. 提交的变更文件列表可独立展开或收起；展开另一条提交时，不得关闭已展开的提交。
2. 每个展开提交独立维护文件数据、加载状态和错误状态，异步请求不能将结果写入其他提交的列表。
3. 提交树的节点、连线和内联文件块必须根据所有已展开块的实际高度对齐，不得重叠或发生纵向错位。
4. 在提交树右上角现有操作按钮区的最右侧提供图标按钮，在平铺列表和树形列表间切换。
5. 树形列表按变更文件的目录层级显示目录和文件；每个目录支持独立展开和收起，嵌套目录的子项随父目录状态显示或隐藏。没有直属文件且仅有一个子目录的连续目录链应按 VS Code Compact Folders 模式压缩为一行。
6. 目录展开状态按提交 hash 与目录路径在当前面板会话内隔离保存；目录初始展开，收起一个提交中的目录不得影响其他提交。
7. 目录收起或展开后，内联列表高度、提交图节点、路径和后续提交行必须使用同一可见行数重新对齐。
8. 将非空内联变更列表的最大高度提高到 `240px`；超出时保留局部滚动。
9. 视图切换对当前已展开的提交立即生效，不关闭列表；视图模式是用户级 renderer-session 偏好，首次加载默认平铺列表，之后重新打开 Git 标签或打开其他项目时沿用最后选择。本任务不新增项目级或跨会话偏好持久化。
10. 从任一模式的文件项打开 diff 时，必须读取该文件所属提交的 diff。

## Acceptance Criteria

- [ ] 展开提交 A 后再展开提交 B，A 与 B 的变更列表同时可见；单独收起 A 不影响 B。
- [ ] 多个展开块存在时，提交图节点、路径和后续提交行与对应内容保持对齐，且列表不被裁切或覆盖。
- [ ] 每个展开块正确展示自身的加载中、错误、空列表和文件列表状态；快速展开/收起不会出现串写结果。
- [ ] 右上角按钮区最右侧存在具备 tooltip、无障碍标签和按下状态的列表/树形切换按钮。
- [ ] 选择树形或平铺后，离开并重新打开 Git 标签或打开另一项目时仍保持该模式；重新加载 renderer 后恢复首次加载的平铺默认值。
- [ ] 树形模式以可展开/收起的目录节点显示嵌套路径，目录具有正确的 `aria-expanded` 状态；没有直属文件的单子目录链显示为一个以 `\` 分隔的紧凑节点；子文件和子目录的前导图标与名称必须在父目录内容之后再缩进一层；平铺模式保留文件名、目录、状态和增删行数。
- [ ] 收起嵌套目录会隐藏其所有后代并立即重新对齐提交图；同一目录在其他展开提交中保持独立状态。
- [ ] 非空变更列表最多展示 `240px` 高度，超出时可在列表内滚动；两种模式下点击文件都可审阅正确提交的 diff。
- [ ] 类型检查、现有测试及与 Git 视图相关的验证脚本通过。

## Out of Scope

- 改变左侧工作区变更列表、提交多选逻辑或 Git 数据桥接协议。
- 为视图模式增加项目级或跨会话持久化。
- 为目录展开状态增加项目级或跨会话持久化。

## Notes

- 该任务涉及异步状态缓存与自绘提交图坐标，规划阶段需要补充技术设计和实现步骤，再进入执行。
