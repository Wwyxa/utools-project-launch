# 重构首页分组搜索布局与刷新动画

## Goal

让首页工具栏优先展示项目分组，同时将低频搜索收纳为按需展开的操作，并消除刷新加载态造成的按钮尺寸跳动。

## Confirmed Facts

- 当前首页搜索框常驻在工具栏第一行，项目分组独占第二行。
- 浏览器中正常按钮组约占 348px，但 uTools 默认缩放与字体取整会产生不同的实际宽度，不能用固定像素或 rem 值代替按钮组的固有宽度。
- 当前刷新按钮在刷新时插入“刷新中”文本并切换内边距，因此加载态会改变按钮宽度。

## Requirements

- 首页工具栏调整为单行的左右两个独立区域。
- 顶部工具栏自身不得成为横向滚动容器；拖动或滚轮操作不能让整条工具栏左右移动。
- uTools 紧凑窗口下缩小工具栏水平内边距和左右区域间距，优先完整容纳固有宽度的按钮区。
- 右侧按钮区域由默认按钮组的实际固有宽度决定，并在搜索框展开或收起时保持该宽度不变。
- 左侧分组区域占据工具栏剩余宽度；分组内容超出可用空间后在区域内部显示细横向滚动条。
- 分组标签继续限制最大文本宽度，过长名称截断，并保留完整名称提示。
- 右侧为按钮区域，默认展示搜索及现有首页操作按钮。
- 点击搜索按钮后，搜索框替换并占满整个右侧按钮区域，不覆盖或挤压左侧分组区域。
- 搜索框出现和关闭时使用轻量过渡动画；动画仅改变透明度和轻微位移，不改变工具栏几何尺寸或触发布局颤抖。
- 展开态不在搜索框末尾添加关闭按钮，避免与原生清空搜索按钮冲突。
- 展开态的搜索图标作为收起按钮；单击工具栏空白处或按 `Esc` 也可收起搜索框。
- 收起搜索框时保留搜索词及对应过滤结果，再次展开可继续编辑。
- 搜索仍按现有规则过滤项目名称、路径和类型，并与当前分组筛选组合生效。
- 首页刷新期间只旋转刷新图标，不显示刷新提示文字。
- 刷新按钮在默认态与刷新态保持相同宽度、高度和位置，同时保留禁用状态及可访问名称。
- 首页项目列表由容器统一管理外部间距：第一行项目与顶部工具栏间距为 12px，紧凑项目区与常规项目区间距为 8px，常规项目行列间距为 8px；项目卡片自身不附加外边距。

## Acceptance Criteria

- [ ] 首页分组与操作按钮位于同一工具栏行，右侧区域宽度等于默认按钮组的实际固有宽度。
- [ ] uTools 默认窗口下外层工具栏不显示横向滚动条，`scrollWidth` 等于 `clientWidth`，且整条工具栏不可左右滑动。
- [ ] 分组区域占据剩余宽度；分组数量或名称过多时，仅分组区域显示细横向滚动条，右侧按钮区域位置稳定。
- [ ] 搜索未展开时显示搜索按钮及现有操作按钮；展开时搜索框占满右侧按钮区域。
- [ ] 搜索切换动画流畅，开关前后的工具栏、分组区域和按钮区域边界保持一致，不出现布局颤抖。
- [ ] uTools 默认窗口缩放下，右侧按钮区域不出现横向滚动条，所有按钮完整可见。
- [ ] 搜索框末尾没有自定义关闭按钮，原生清空搜索控件可正常使用且不被遮挡。
- [ ] 展开态可通过搜索图标、工具栏空白处或 `Esc` 收起，收起后搜索词和过滤结果保持不变。
- [ ] 展开和收起搜索框不会改变左右区域宽度。
- [ ] 搜索过滤、分组过滤以及两者组合后的项目结果与现有逻辑一致。
- [ ] 刷新过程中不出现可见提示文字，刷新图标持续旋转。
- [ ] 刷新前后按钮外部尺寸不变，工具栏中的相邻按钮不发生位移。
- [ ] 首页第一行项目距工具栏底部 12px，紧凑与常规项目区间距为 8px，常规项目网格行列间距为 8px，项目卡片计算样式的外边距为 0。
- [ ] 搜索与刷新控件均保留可访问名称和键盘可操作性。
- [ ] `npm run lint` 与 `npm run build` 通过，并在浏览器中检查常规及紧凑窗口下无工具栏溢出或控件重叠。

## Out of Scope

- 修改搜索匹配字段或筛选结果语义。
- 调整首页其他弹窗、项目卡片或全局刷新逻辑。
- 修改非首页位置的刷新按钮。

## Notes

- 本任务属于单组件内的轻量级交互与布局调整，采用 PRD-only 规划。
