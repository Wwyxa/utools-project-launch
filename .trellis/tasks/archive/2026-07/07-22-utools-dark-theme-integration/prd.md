# uTools 暗色主题融合优化

## Goal

优化插件在 uTools 暗色界面中的视觉融合度，使主界面的背景、层级、边框、文字和交互控件与宿主环境协调，同时保持项目启动器现有的信息密度与可辨识度。

## Background

- 插件已在先前任务中优化过暗色模式，但当前效果与 uTools 暗色宿主仍有明显割裂感。
- 用户提供的“兔灵截图工具”暗色设置页在色彩、表面层级和控件风格上与 uTools 融合良好，可作为视觉方向参考，但不要求逐像素复刻。
- 2026-07-20 的暗色优化任务明确选择了 GitHub 式“中性偏冷石墨灰”；当前 `src/index.css` 暗色令牌实际使用 `#171c23`、`#20262e`、`#2a323c` 等偏蓝灰色，解释了它与 uTools 中性灰宿主之间的色温差异。
- 当前 `src/App.vue` 的自动主题已经优先调用 `window.utools.isDarkColors()`，并以 `prefers-color-scheme` 作为浏览器预览回退；无需新增主题检测依赖或状态层。
- uTools 官方窗口 API 文档提供 `utools.isDarkColors(): boolean`，并推荐使用 `matchMedia('(prefers-color-scheme: dark)')` 监听主题变化；官方事件文档确认 `utools.onPluginEnter(callback)` 会在每次进入插件时触发。
- 当前 `onPluginEnter` 回调只恢复项目与运行状态，没有重新执行 `updateTheme()`；当 uTools 主题在插件隐藏期间变化时，现有自动主题可能保持旧值。
- 第一轮中性化后的真实 uTools 截图显示，插件外层 `#242424` 虽已消除蓝灰色温，但仍明显暗于约 `#333333` 的宿主顶栏，整体观感偏暗沉，尚未完全融入宿主。
- 用户实测 uTools 暗色模式底色为 `#303133`；插件顶部的分组/按钮区使用 `#333333`，项目列表主区应使用 `#303133` 与宿主底色精确衔接。
- 用户补充的“快命令”插件参考表明，融合良好的暗色界面可在宿主底色上使用低饱和冷灰抬升面、克制的普通按钮，以及仅在选中项上出现的高识别度填充和细侧边强调；本插件参考这种层级方法，不复制其蓝色品牌色或大圆角布局。

## Requirements

- R1：优化覆盖插件全部主要界面，重点校准仪表盘、项目详情、设置、环境、弹层及浮动菜单。
- R2：页面底色保持 uTools 的 `#303133`，内容表面、悬浮状态和浮层采用轻微冷调、低饱和的抬升灰阶；边框只辅助分层，不形成一圈圈发亮描边。
- R3：统一顶部工具栏、筛选标签、项目卡片、按钮、输入控件和弹层的暗色视觉语言；普通按钮保持安静，选中/聚焦/运行态使用更明确的强调填充或既有侧边标记。主文字与图标使用柔和冷灰而非纯白，次要文字和状态色保持可辨但不刺眼。
- R4：自动主题继续结合 uTools 暗色检测与标准媒体查询，并在每次进入插件时重新同步宿主主题。
- R5：保留现有绿色品牌/运行状态语义，用绿色承担参考图中蓝色高亮的功能角色；不复制参考插件的品牌蓝色，并避免大面积高饱和填充。
- R6：保留现有功能、布局结构、信息密度、项目状态表达和交互反馈。
- R7：浅色模式不得因本次调整发生明显视觉回退。
- R8：优先复用现有 CSS 变量与平台能力，不新增仅用于换色的依赖、状态层或第二套主题系统。

## Acceptance Criteria

- [ ] AC1：在 uTools 暗色模式中打开插件时，项目列表主区使用宿主底色 `#303133`，顶部区保持 `#333333`，外层不再呈现色温或明度断层，整体观感不暗沉。（R1、R2）
- [ ] AC2：自动主题下，在插件隐藏期间切换 uTools 明暗主题后重新进入插件，根主题能同步到当前宿主主题；显式浅色/暗色不被覆盖。（R4）
- [ ] AC3：仪表盘、项目详情、设置、环境、弹层及浮动菜单使用统一的低饱和冷灰抬升面；边框和阴影不抢占内容注意力，页面底色仍与宿主连续。（R1、R2、R3）
- [ ] AC4：普通按钮视觉克制；悬浮、按下、选中、聚焦、禁用、错误和运行中状态清楚区分，绿色选中/运行态比默认面更明确但不刺眼。（R3、R5、R6）
- [ ] AC5：正文、图标、辅助信息和弱化信息在暗色背景下可读，不使用纯白作为常规或高对比前景，且不依赖高亮描边维持辨识度。（R3）
- [ ] AC6：主要界面在正常桌面尺寸与紧凑 uTools 类尺寸下无文字溢出、控件遮挡或布局跳动。（R1、R6）
- [ ] AC7：浅色模式的现有页面和关键交互通过回归检查，代码预览与语法配色不发生变化。（R7）
- [ ] AC8：实现只复用现有主题解析与语义令牌，不新增换色依赖、重复主题状态或组件级调色板。（R8）
- [ ] AC9：`npm run lint` 与 `npm run build` 通过。（R6、R7、R8）

## Out Of Scope

- 不重新设计项目启动器的信息架构或业务流程。
- 不照搬参考插件的品牌色、图标、布局或文案。
- 不为本次视觉调整引入完整的第三方主题框架。
- 不重新设计代码预览、diff 或 Markdown 代码块的语法高亮配色。
