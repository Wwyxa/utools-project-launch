# 丰富 Files Tab 功能

## Goal

在不增加重量级依赖的前提下，丰富项目详情页 Files Tab 的文件浏览与操作能力，并让 Markdown 预览正确显示项目内本地图片。

## Background

- 当前 Files Tab 支持文本/代码（含 Markdown 渲染和编辑）与常见图片预览，其他二进制文件显示不可预览提示。
- 当前文件树只有懒加载展开、单击预览和双击编辑，没有刷新、折叠、名称过滤、键盘树导航、节点菜单或文件管理操作。
- 当前 Markdown 渲染保留图片相对 `src`，浏览器会相对插件页面 URL 请求资源，而不是相对当前 Markdown 文件目录解析；例如 README 中存在的 `docs/screenshots/Git状态.png` 因而无法显示。
- Flyfish Viewer 2.1.30 的 Office 与 PDF-only 组合经隔离安装和生产构建验证后均超过本项目的轻量预算，因此决定不接入且不再扩展预览格式；许可证、支持范围和体积证据见 `research/flyfish-viewer-evaluation.md`。

## Requirements

- 保留现有文件树、文本编辑、查找替换、代码高亮、Markdown 和图片预览行为。
- 不接入 Flyfish Viewer，不新增文件预览类型或相关运行依赖。
- Files Tab 渲染 Markdown 时，项目内本地图片引用必须基于当前 Markdown 文件位置解析，通过既有项目文件读取边界加载；远程 URL、`data:` URL 和普通 Markdown 文本保持原行为。
- 本地图片路径支持 `./`、`../`、裸相对路径和以 `/` 开头的项目根路径；拒绝盘符绝对路径、UNC、`file:` URL 及任何越过项目根目录的引用。
- 本地图片加载必须复用项目目录越界检查和现有图片大小/类型限制；不存在、不支持、超限或越界的资源不得导致整个 Markdown 预览失败。
- Markdown 正文图片必须限制在预览内容宽度内并保持原始宽高比，避免大图撑破右侧面板。
- 文档开头存在闭合 YAML front matter 时，将元数据作为 YAML 高亮块展示，长行随预览宽度折行且不显示水平滚动条；后续标题、段落、行内代码和列表继续按 Markdown 正常渲染，普通 `---` 分隔线不得被误判。
- 新增 Files Tab 能力必须沿用项目根目录路径边界校验，并对失败操作提供明确反馈。
- 增加导航效率能力：刷新文件树、全部折叠、按文件/目录名过滤、键盘导航、复制相对路径、复制绝对路径、在系统文件管理器中显示。
- 名称过滤递归覆盖整个项目目录，沿用现有忽略目录规则；结果必须设置数量上限并显示相对路径，同名结果可区分，扫描不得阻塞其他文件操作。
- 增加文件管理能力：新建文件、新建目录、重命名和删除文件或目录。
- 新建目标按当前选择推导：选中目录时创建在该目录内，选中文件时创建在其父目录，未选中节点时创建在项目根目录；目录上下文菜单中的新建操作始终以该目录为目标。
- 新建和重命名使用树内就地输入；Enter 提交、Escape 取消，非法名称、重名和操作失败在输入位置显示，失败时保留输入值供用户修正。
- 新建文件成功后立即打开并进入编辑状态；新建目录成功后展开并选中。
- 顶部工具栏承载新建、刷新、全部折叠和名称过滤；节点级重命名、删除、复制路径和系统定位通过上下文菜单提供，上下文菜单必须支持鼠标右键与键盘打开。
- 文件节点上下文菜单宽度按中英文实际内容收缩，并在渲染后使用真实尺寸定位；靠近视口右侧或底部时应围绕触发点翻转且保持安全边距。
- 文件系统写操作必须处理重名冲突、无效名称、项目根目录越界和操作失败，删除必须经过明确确认。
- 切换、重命名或删除受当前未保存草稿影响的文件时，统一提供“保存 / 放弃 / 取消”三种选择；确认对话框必须复用项目既有样式和交互模式。
- 允许递归删除非空目录；危险确认必须显示目标目录名，并明确提示其包含内容会一并删除，执行期间禁止重复提交。

## Acceptance Criteria

- [x] 已记录候选库支持格式、许可证、直接与传递依赖以及实际构建体积，并决定不接入。
- [x] Files Tab 首期范围确定为导航效率与文件管理。
- [ ] README 中的 `![Git 状态](docs/screenshots/Git状态.png)` 可以显示现有 `docs/screenshots/Git状态.png`。
- [ ] 嵌套目录 Markdown 的 `./`、`../` 和裸相对图片路径基于 Markdown 文件所在目录正确解析，越界引用被拒绝。
- [ ] `/assets/example.png` 按项目根目录解析；盘符、UNC、`file:` 和项目外路径不会被读取。
- [ ] 本地图片不存在、不支持或超限时仅该图片显示失败状态，Markdown 其他内容继续渲染；远程与 `data:` 图片不被改写。
- [ ] Markdown 图片在窄面板中不产生横向溢出并保持宽高比。
- [x] YAML front matter 显示为无水平滚动条的高亮元数据，长行随容器折行，正文结构继续正常渲染，普通水平分隔线保持原语义。
- [ ] 用户可以刷新/折叠/过滤文件树，并通过键盘完成主要树导航。
- [ ] 名称过滤可以找到尚未展开目录中的匹配项，不扫描既有忽略目录，达到结果上限时明确提示。
- [ ] 全局操作可从顶部工具栏发现，节点操作可通过鼠标或键盘打开同一个上下文菜单完成。
- [x] 文件节点菜单按内容自适应宽度，并在视口四角使用真实菜单尺寸完成翻转定位。
- [ ] 用户可以复制节点路径或在系统文件管理器中定位节点。
- [ ] 用户可以新建、重命名和删除文件/目录，操作成功后文件树和当前预览状态保持一致。
- [ ] 新建文件/目录默认落在当前选择推导出的目录，成功后目标父目录展开并选中新节点。
- [ ] 新建文件成功后右侧立即进入该文件的编辑状态；新建目录成功后目录保持选中和展开。
- [ ] 新建与重命名可完全通过键盘完成；失败时输入内容不丢失，用户可以修正后重试或取消。
- [ ] 用户确认后可以递归删除非空目录；取消或失败时磁盘和前端树状态保持不变。
- [ ] 无效、冲突、越界或失败操作不会留下错误的前端树状态，并显示明确反馈。
- [ ] 当前文件存在未保存编辑时，切换、重命名或删除不会静默丢失内容，用户可选择保存、放弃或取消。
- [ ] 文本编辑、查找替换、Markdown、代码高亮和图片预览无回归。
- [ ] 类型检查与生产构建通过。

## Out of Scope

- 新增任何文件预览格式。
- Flyfish Viewer 或其他文档预览库接入。
- 云端转换、远程预览服务或文件上传。
