# UI改造优化

## Goal

围绕项目列表、项目详情/终端、编辑弹窗和设置页进行一轮偏开发者工具气质的 UI 改造：降低噪音、强化运行态信息、提升多脚本项目的启动效率，并优化配置录入体验。

追加目标：将首页从偏展示型卡片回调为高效控制面板。保留新版整洁视觉、技术标签、脚本 Dropdown 收纳等功能，但显著提高项目列表的信息密度、列数和一屏可见项目数量。

第三轮追加目标：继续对首页卡片做减法，删除重复的技术栈/类型行，把环境信息合并进标题右侧徽标，并将阅读流调整为项目名 -> 描述 -> 路径 -> 脚本。

第四轮追加目标：修复首页卡片微观排版粗糙点，尤其是脚本按钮文本截断/拉伸、暗色 hover 操作区突兀、同排卡片底部不齐、路径层级不够弱的问题。

第五轮追加目标：统一卡片脚本区与底部分隔线之间的呼吸节奏，并将主界面项目排序从上下按钮升级为拖拽排序。

## What I Already Know

- 项目列表卡片实现位于 `src/components/dashboard/ProjectCard.vue`，当前路径已截断但状态标签会在停止态也显示，底部操作按钮默认常驻。
- 项目大盘位于 `src/components/dashboard/Dashboard.vue`，卡片采用响应式 grid。
- 项目详情位于 `src/components/project/ProjectDetails.vue`，概览中即使没有 git 快照也会显示默认 `main` 分支。
- 脚本列表和终端位于 `src/components/project/ScriptsTab.vue` 与 `src/components/terminal/Terminal.vue`，当前已支持单脚本启动/停止和按脚本日志标签。
- 编辑弹窗位于 `src/components/project/ProjectFormModal.vue`，脚本排序当前使用上下箭头，项目路径检查已能自动识别 package/python 脚本。
- 设置页位于 `src/components/layout/SettingsTab.vue`，自定义终端命令已有独立输入样式。
- 状态与桥接层位于 `src/store/useStore.ts`、`src/lib/projectBridge.ts`、`public/preload.js`；批量启动可复用 `launchScript`，工作目录提示需要新增读取子目录的 bridge/store 能力。
- 项目使用 Vue 3 + Pinia + Vite + Tailwind CSS v4 + lucide-vue-next；质量命令是 `npm run lint`，即 `tsc --noEmit`。

## Assumptions

- 这次优先做现有技术栈内的轻量实现，不引入新的大型 UI 组件库。
- 终端 ANSI 颜色优化以当前 `LogEntry.type` 和文本模式启发式分级为主；不实现完整 ANSI escape parser，除非后续确认需要保留原始 ANSI 控制码。
- 拖拽排序可使用原生 HTML5 drag/drop 完成，避免新增依赖。

## Requirements

### 项目列表页

- 首页布局必须回归紧凑多列控制台：根据窗口宽度优先展示 3 列，宽屏可展示更多列，避免两列极宽卡片造成大量留白。
- 项目名称保持主视觉焦点；绝对路径使用单行截断、小字号、次级灰显示。
- 卡片第一行显示项目名称 + 技术栈徽标；技术栈和环境标识必须合并到标题右侧徽标中，例如 `Node.js`、`Go`、`Python | uv`。
- 删除任何单独的技术栈/类型展示行，例如 `< /> Node.js` 这类额外标签行不应再出现。
- 描述信息优先级高于物理路径：如果项目有描述，描述显示在标题下方第二行；无描述时不占空间。
- 路径只允许显示一次，位于描述之后，作为最低优先级补充信息；颜色要更暗更克制，并优先展示尾部路径或中间省略形式，避免从盘符开始的长路径占据视觉焦点。
- 停止态不显示灰色“已停止”状态点/标签；仅运行态显示绿色“运行中”，错误态保留必要错误提示。
- 运行/错误状态要足够醒目，可放在卡片右上角或底部信息旁；停止态保持克制，但整体状态层级不能只依赖项目名称颜色。
- 卡片右下角终端、文件夹、编辑、删除操作默认隐藏，仅 hover/focus 时平滑显示；排序模式的排序按钮保持可见。
- 隐藏脚本数量从 `+N` 改为带下拉箭头的 `...` 控件，展示剩余脚本菜单并可启动/停止脚本。
- 隐藏脚本必须使用点击触发的 Dropdown 菜单，不应只是 hover title/临时浮层；菜单列表要明确展示脚本名，点击可直接运行/停止。
- 卡片内容使用纵向 flex，使底部更新时间和操作区吸底对齐。
- 卡片内部结构保持紧凑：首行项目名称 + 技术栈/环境徽标，第二行可选描述，第三行低优先级路径，第四行紧凑脚本区；无描述时不占空间。
- 缩小脚本按钮间距和脚本区与正文的间距，降低卡片高度，提高一屏可见项目数量。
- 脚本区默认只展示 2-3 个最高频/最重要命令，剩余脚本全部收进 `...∨` Dropdown，尽量保持脚本区一行显示。
- 脚本按钮文字绝对不能在按钮内部截断成 `front...` / `backe...`；外露按钮必须完整可读。
- 脚本按钮宽度由内容和 padding 决定，不能自动拉伸填满卡片宽度；所有脚本按钮左对齐，右侧留白可接受。
- 当一行无法容纳完整按钮时，优先将多余的整个按钮收进 `...∨` Dropdown，而不是截断按钮文字。
- 项目技术栈标签增加更明确的微型标识/暗色调区分；Python 可在标签中弱化展示 `uv/conda` 等环境提示（可先基于脚本/备注/环境变量启发式识别）。
- hover 时卡片有轻微背景提亮或边框 glow。
- 运行中状态必须语义明确：可保留绿色边框，但还需要右上角或标题旁的绿色呼吸点或微型 `Running` 文案。
- 底部操作图标默认必须隐藏，只在卡片 hover/focus-within 或排序模式下显示。
- 底部操作图标不得使用暗色模式下刺眼的浅色容器背景；默认仅低透明度显示/隐藏，卡片 hover 后提升透明度，单个图标 hover 时只使用极弱背景。
- 卡片主体应使用 flex/space-between 结构，确保同一排卡片的更新时间/底部区域压到底部并水平对齐。
- 卡片脚本区与底部分隔线之间的距离要保持一致；无论脚本按钮数量多少，按钮不应紧贴分隔线。
- 路径文字层级必须低于描述，颜色更暗、字号更小，作为低注意力的辅助信息。
- 主界面项目排序应支持拖拽排序；进入排序模式后通过拖动卡片或拖拽手柄调整项目顺序，替代上下/置顶按钮式排序。
- 顶部搜索和工具栏保持现有功能，搜索框可略微增加高度并强化 focus 状态。

### 项目详情与终端日志

- 若没有检测到 `.git` 仓库快照，则概览中隐藏/替换分支信息，避免左侧显示 `main`、右侧显示无仓库的冲突。
- 多脚本项目增加 `Run All` 总控按钮，启动所有未运行且命令非空的脚本。
- 终端区域使用更深背景（目标接近 `#0D1117`）和轻微内阴影。
- 日志颜色区分：时间戳置灰，INFO 绿色/青色，WARNING 黄色，ERROR 红色，普通输出保持低对比终端前景。
- 正在运行且有日志的标签显示微型绿色脉冲/呼吸灯。

### 编辑弹窗与设置页

- 脚本列表废弃上下箭头排序，改为左侧拖拽手柄并支持拖拽排序。
- 编辑项目弹窗表单项增加垂直间距，整体更有呼吸感。
- 设置页自定义命令输入框与全局输入框圆角、边框、focus 表现统一。
- 添加/编辑启动命令时，工作目录输入框提供下拉建议；读取项目根目录下一层子目录，并将 `frontend`、`backend`、`client`、`server`、`api`、`src` 等常见目录优先排序。

### uTools 生态

- 支持从 uTools 主输入框按项目名称搜索项目，并按回车直接进入该项目的启动/详情区。

## Acceptance Criteria

- [ ] 项目卡片停止态不再显示灰色状态标签，运行态仍醒目可见。
- [ ] 首页在常见 uTools 窗口宽度下恢复 3 列或更多列的紧凑控制台布局，不再出现两列极宽卡片。
- [ ] 项目路径只显示一次，且位于项目名称下方，描述与脚本区不再出现冗余路径/标签区。
- [ ] 卡片不存在独立的 `< /> Node.js` 等技术栈/类型行；环境信息只并入标题右侧徽标。
- [ ] 卡片信息顺序为项目名/徽标、可选描述、低优先级路径、脚本区。
- [ ] 卡片操作按钮只在 hover/focus 或排序模式下显现，底部信息在不同描述长度下保持对齐。
- [ ] 超出显示数量的脚本通过 `...▼` 轻菜单查看并可操作。
- [ ] 脚本 Dropdown 为点击触发菜单，菜单项可直接运行/停止脚本。
- [ ] 卡片高度与脚本区间距足够紧凑，一屏可见项目数量较上一版提升。
- [ ] 外露脚本按钮完整显示文字，不出现按钮内部省略号截断。
- [ ] 单个 `run` 等短脚本按钮不会拉伸填满卡片宽度，所有脚本按钮统一左对齐。
- [ ] 运行中卡片既有边框/强调，也有绿色呼吸点或微型 Running 文案，语义清晰。
- [ ] 暗色模式下 hover 操作图标没有刺眼浅色背景块。
- [ ] 同一排卡片底部更新时间/操作区域视觉上对齐。
- [ ] 各卡片脚本区与底部分隔线之间保持一致的空隙。
- [ ] 路径文字明显弱于描述文字。
- [ ] 主界面项目列表支持拖拽排序，并通过现有 store 排序/持久化逻辑保存顺序。
- [ ] 项目详情无 git 快照时不显示虚假的 `main` 分支信息。
- [ ] 脚本页提供 Run All，并能批量启动未运行脚本。
- [ ] 终端日志具备深色终端质感、分级颜色和运行标签 pulse。
- [ ] 编辑弹窗脚本支持拖拽排序，移除上下排序按钮。
- [ ] 工作目录输入框可展示项目子目录建议。
- [ ] uTools 主输入框可按项目名称匹配并进入对应项目详情/启动区。
- [ ] `npm run lint` 通过。

## Definition of Done

- 修改保持在现有 Vue/Pinia/Tailwind/lucide 栈内。
- TypeScript 类型同步更新，桥接 fallback 与 preload 两端一致。
- 不破坏浏览器预览 fallback。
- 必要时更新 Trellis spec 或记录无需更新的判断。

## Out of Scope

- 完整终端 ANSI escape parser。
- 新增大型拖拽库或 UI 库。
- 重新设计整体信息架构或新增独立页面。
- Git 功能深度改造。
- 全文搜索引擎或复杂模糊搜索算法；本任务只做项目名称直达。
