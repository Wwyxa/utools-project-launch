# 整体细节调整

## Goal

优化项目启动器的日常使用细节：让首页更清楚地展示正在运行的具体命令，删除当前无实际意义的侧边栏/操作按钮，调整 uTools 插件窗口尺寸策略，并补齐项目管理中的删除能力。

## What I already know

* 用户希望本次任务先聚焦整体细节，不新增大型功能模块。
* 首页项目卡片目前只展示项目级运行状态，未直接说明具体哪个启动命令正在运行。
* 项目脚本数据已有 `script.status === "RUNNING"`，可以从现有数据推导运行中的命令。
* 侧边栏当前包含项目、插件、备忘、设置、文档和新增项目入口，其中插件、备忘、文档入口当前缺少对应有效主视图或明确行为。
* `public/plugin.json` 当前在 `pluginSetting` 中设置了固定 `height: 760`，用户反馈默认窗口过大，希望参考 uTools 文档，避免不必要的固定尺寸。
* 项目表单支持新增/编辑，store 当前没有删除项目的 action，UI 中也没有项目删除入口。

## Assumptions (temporary)

* 首页展示运行命令时优先显示脚本名称，其次显示命令文本；多个命令运行时显示前几个并给出数量提示。
* “无意义按钮”优先删除没有可用目标或没有实际行为的入口，而不是保留 disabled/coming soon 状态。
* 删除项目先做本地状态删除：移除项目、日志、Git 暂存数据、待办和备忘缓存；不删除磁盘目录。
* 删除操作需要二次确认，避免误删项目配置。

## Open Questions

* 暂无阻塞问题。

## Requirements (evolving)

* 首页项目卡片在项目运行中时必须明确展示正在运行的命令/脚本，而不是只显示“运行中”。
* 运行命令展示不能破坏卡片布局；长命令需要截断或换行处理。
* 删除侧边栏中当前无实际页面或有效行为的按钮，保留稳定可用的核心导航和新增项目入口。
* 侧边栏设置入口应放在侧边栏底部；项目按钮下方预留/承载真正有用的功能性 tab 控制入口。
* 功能性 tab 入口需要围绕项目启动器的核心工作流考虑，避免恢复无意义占位按钮。
* 清理项目卡片或其他位置中点击后无明确效果的无意义按钮。
* 按 uTools 插件配置建议调整窗口尺寸策略，避免当前默认窗口过大；如无必要，不设置固定大高度。
* 项目管理支持删除项目配置，并同步清理相关内存数据。
* 删除项目入口同时放在项目详情页主操作区和首页卡片 hover 操作区。
* 删除项目后，如果当前正在查看该项目，应返回项目列表；如果删除的是列表中的项目，不应留下失效选中状态。
* 删除能力应提供清晰的确认文案和可取消路径。
* 项目卡片不应使用额外的“正在运行”信息卡/大块区域；运行中的命令应在脚本标签本身用高亮、状态点或类似紧凑样式表达。
* 项目卡片需要进一步压缩高度和空白，尤其减少底部时间信息与周围留白占用。
* 项目卡片右侧空间需要承载状态/操作等有效信息，避免内容集中在左侧导致右侧大片空白。
* 项目卡片展示方式应更接近启动器/管理工具的紧凑列表项，而不是松散展示卡片。
* 项目卡片操作应围绕高频项目管理动作：在终端中打开、打开目录、编辑、删除；不需要专门的查看详情和刷新 Git 按钮。
* 项目卡片应以命令启动效率为核心，启动命令标签可以直接点击执行或停止，减少进入详情页的步骤。
* 项目卡片不应使用笨重的右侧操作列；管理动作应轻量呈现，避免破坏卡片整体美观。
* 项目运行状态应固定在卡片右上角，避免和项目类型标签挤在同一标题流中导致换行。
* 项目卡片内容不应因为固定右侧预留空间过大而过早省略，文本应尽量使用卡片可用宽度。
* 暗色模式下避免亮绿色 `#63DCA6` 背景搭配白色文字/图标，设置页、项目详情页等主色按钮也要使用更柔和且可读的前景色。
* 项目数量较多时，页面滚动应保持顶部搜索、刷新、新增等按钮置顶。
* 暗色模式下右侧滚动条样式不能突兀，应与主题颜色协调。
* 暗色模式下顶部搜索、刷新、新增等工具栏背景不能出现亮色 `#EFEDEC` 一类浅色块，应跟随暗色主题语义色。
* 首页顶部工具栏需要更紧凑，不显示与项目列表之间明显的分隔条。
* 侧边栏新增项目按钮多余，应移除；新增项目入口保留在首页顶部工具栏。
* 设置页布局需要提高信息密度，避免几个 card 摆放过低效和大量空白。
* 设置页增加默认终端配置，用于后续项目在终端中打开或类似启动用途。

## Acceptance Criteria (evolving)

* [ ] 当项目中某个脚本为 `RUNNING` 时，首页卡片能看到对应脚本名/命令提示。
* [ ] 当多个脚本运行时，首页卡片能表达多命令运行状态。
* [ ] 侧边栏不再出现没有落地页面或无实际行为的插件、备忘、文档类按钮。
* [ ] 设置按钮位于侧边栏底部，项目按钮下方只放有明确功能价值的 tab 控制按钮。
* [ ] 项目卡片不再包含无效点击按钮。
* [ ] 运行中的脚本标签在卡片内以紧凑高亮样式表达，无额外大块运行中区域。
* [ ] 项目卡片整体更紧凑，底部时间/操作区域不再占用过多高度。
* [ ] 项目卡片右侧不再显得空，状态和常用操作占据右侧空间。
* [ ] 项目卡片提供在终端中打开按钮，不再显示查看详情和刷新 Git 按钮。
* [ ] 项目卡片上的启动命令可直接点击启动/停止。
* [ ] 项目卡片操作区轻量美观，不再是低效的右侧按钮列。
* [ ] 项目运行状态位于卡片右上角，不影响项目类型标签展示。
* [ ] 项目名称、路径和描述不会因过大的右侧预留空间过早截断。
* [ ] 暗色模式下主色按钮不再使用刺眼的亮绿底白字/白图标组合。
* [ ] 项目列表滚动时顶部搜索、刷新、新增等控制保持置顶。
* [ ] 暗色模式下滚动条样式与界面主题协调，不出现突兀亮色滚动条。
* [ ] 暗色模式下首页顶部搜索、刷新、新增工具栏背景与控件背景均为协调暗色。
* [ ] 首页顶部工具栏高度更紧凑，且与项目列表之间没有明显分隔条。
* [ ] 侧边栏不再显示新增项目按钮。
* [ ] 设置页 card 布局更紧凑，减少空白。
* [ ] 设置页提供默认终端配置项。
* [ ] 插件配置不再强制使用当前过大的默认固定高度。
* [ ] 用户可以从项目详情页删除项目配置，删除前有确认，删除后列表和详情状态正确更新。
* [ ] 用户可以从首页卡片 hover 操作区删除项目配置，删除前有确认，删除后列表状态正确更新。
* [ ] lint/type-check 通过。

## Definition of Done (team quality bar)

* Tests added/updated when appropriate for changed behavior.
* Lint / typecheck green.
* Docs/notes updated if behavior changes.
* Rollout/rollback considered if risky.

## Out of Scope

* 不实现真实持久化存储迁移或云同步。
* 不删除用户磁盘上的项目目录。
* 不新增插件、备忘等独立一级页面。
* 不重做整体视觉主题或布局体系。
