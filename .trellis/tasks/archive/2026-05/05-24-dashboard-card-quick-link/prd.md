# 优化首页排序卡片与快捷访问链接

## Goal

优化项目首页卡片在排序模式下的视觉稳定性，并为项目增加可配置的快捷访问链接。用户启动部分项目后常需要打开一个本地或远程页面；该地址不一定能从日志可靠识别，因此本任务提供手动配置入口，并在首页卡片中按需显示快捷访问按钮。

## What I Already Know

- 首页项目卡片由 `ProjectCard.vue` 渲染，排序开关位于 `Dashboard.vue`。
- 当前卡片底部操作区在非排序模式下包含终端、编辑器、目录、编辑、删除等 hover 操作；排序模式下操作区隐藏内容但保留底部布局。
- 项目数据类型和表单值定义在 `types.ts`，项目持久化、表单草稿、导入导出和 demo 数据由 Pinia store 管理。
- 项目表单已有基础信息、图标、脚本、环境变量、备忘等配置区域，可新增项目级 URL 字段。

## Requirements

- 首页进入排序模式后，项目 card 高度不应因为排序状态、拖拽手柄或操作区变化而改变。
- 新增一个项目级快捷访问链接配置字段，支持创建和编辑项目时填写。
- 快捷访问链接为空时，首页卡片不显示快捷访问按钮，也不预留按钮空位。
- 快捷访问链接有值时，首页卡片右上角显示一个紧凑的图标按钮，用于打开该链接。
- 快捷访问按钮不能让卡片标题、状态、排序手柄、错误状态等元素拥挤或重叠。
- 链接打开失败或链接为空时不应影响卡片选择、脚本启动、排序拖拽等已有交互。
- 新增字段需要随项目持久化、编辑表单回填、导入导出一起保留。
- 中英文文案保持一致，使用现有 i18n 模式。

## Acceptance Criteria

- [ ] 开启/关闭首页排序后，相同项目 card 的高度保持一致。
- [ ] 创建或编辑项目时可以填写快捷访问链接。
- [ ] 未配置快捷访问链接的项目卡片右上角不显示快捷访问按钮，布局不出现无效空白。
- [ ] 已配置快捷访问链接的项目卡片右上角显示按钮，点击后打开该 URL，且不会触发卡片详情选择。
- [ ] 排序模式下快捷访问按钮不干扰拖拽排序。
- [ ] 项目保存、重新加载、导入导出后快捷访问链接仍然保留。
- [ ] `npm run lint` 和 TypeScript 检查通过。

## Out of Scope

- 自动从启动日志中识别 URL。
- 为每个脚本单独配置多个快捷访问链接。
- 校验服务是否已启动或链接是否可访问。
- 新增浏览器选择、内置预览或链接健康检查。

## Technical Notes

- 优先使用已有 `bridge.openPath` 打开 URL，避免新增 preload API。
- 项目级字段建议命名为 `quickLink` 或同等语义字段，并在 hydrate/persist/form/import 路径中统一处理。
- 卡片右上角按钮应为图标按钮，使用 lucide 图标和 title/aria-label；无配置时不渲染。
- 卡片排序高度问题应从布局稳定性处理，避免通过增加无意义空白解决。
