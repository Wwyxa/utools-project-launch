# 05-18-ui-refinement: UI 完善与改进

## Goal

优化 uTools 插件的整体 UI 设计，使其更符合 uTools 插件的使用习惯（紧凑、简洁、减少文字冗余）。主要改进侧边栏、顶栏以及界面布局，提升交互体验。

## What I already know

- **界面风格**: 保持简洁，但需适应 uTools 窗口较小的特性，优化元素大小和排版。
- **减少冗余**: 减少直接的文字说明，避免大片空白。
- **侧边栏 (Sidebar)**:
  - 完全收起，不需要文字标签。
  - 移除中英切换按钮。
  - 移除退出按钮。
  - 点击非“项目”tab 需要有相应反应（目前无反应）。
- **顶栏 (TopBar)**:
  - 移除“项目管理器”标题/介绍。
  - 移除或确认右上角通知按钮的意义（无意义则删除）。
  - 考虑移除独立于 tab 的顶栏。

## Assumptions (temporary)

- 侧边栏收起后仅显示图标。
- 点击侧边栏不同 Tab 应该切换到对应的功能页面（即使是占位符）。
- 通知按钮目前没有具体功能需求，倾向于删除。

## Open Questions

- 当侧边栏完全收起且没有独立顶栏时，原本在顶栏的操作（如搜索、添加项目等）应该如何重新排版？
- 目前侧边栏除了“项目”外，其他 Tab（如“插件”、“设置”）的具体预期功能是什么？

## Requirements (evolving)

- [x] 优化侧边栏样式：完全收起，仅显示图标，移除退出和语言切换。
- [x] 移除顶栏标题和不必要的通知按钮。
- [x] 实现侧边栏所有 Tab 的基本切换逻辑。
- [x] 调整全局布局，适应紧凑窗口，减少空白。
- [ ] 实现亮色、暗色主题切换，支持跟随系统。

### 具体修改点

1. **Store 层**:
   - 添加 `activeTab` 状态（'projects', 'plugins', 'memos', 'settings'）。
   - 初始值为 'projects'。
   - 点击侧边栏项时更新 `activeTab`。
   - 如果 `selectedProjectId` 存在，通常视为在 'projects' tab 的详情页。

2. **Sidebar.vue**:
   - 宽度从 220px 缩减为固定宽度（如 64px）。
   - 移除标题、副标题、退出按钮、语言切换按钮。
   - “新建项目”按钮改为只有图标的圆形按钮。
   - 导航项只显示图标，移除文字。
   - 确保所有 Tab 点击都能切换 `activeTab`。

3. **App.vue**:
   - 调整 `ml-[200px]` 为新的侧边栏宽度（如 `ml-[64px]`）。
   - 根据 `activeTab` 渲染不同的组件。
   - 考虑移除全局 `TopBar`，或将其逻辑移入具体页面。

4. **TopBar.vue / Header**:
   - 移除“项目管理器”文字。
   - 移除通知按钮。
   - 如果保留，只保留搜索、语言（如果需要，但侧边栏已移除，可以考虑放在设置页或顶栏角落）、刷新等功能。
   - 用户建议“不需要有一个独立于各tab的顶栏”，这意味着顶栏的内容（如搜索）应该整合进 `Dashboard.vue` 或具体页面中。

5. **界面细节**:
   - 减小元素间距。
   - 移除冗余说明文字。

### 最终确认逻辑

1. **移除 TopBar.vue**:
   - 全局移除 `TopBar` 组件的使用。
   - 将“项目搜索”功能移入 `Dashboard.vue` 列表顶部。
   - 将“项目刷新”功能移入 `ProjectDetails.vue` 或 `Dashboard.vue`。
   - 侧边栏保持极简（只有图标），语言切换移入“设置”Tab 页面中（不在侧边栏常驻）。

2. **Sidebar 极简化**:
   - 宽度固定 64px。
   - 移除所有文字标题。
   - 退出按钮移除。
   - “新建项目”变成一个圆形的 Plus 图标按钮。

3. **布局调整**:
   - `App.vue` 移除 `TopBar`。
   - 主内容区域占满剩余空间。
   - 减小 Dashboard 中的卡片间距和外边距。

4. **主题切换 (Theme)**:
   - 参考 uTools 文档，支持亮色 (Light)、暗色 (Dark) 和跟随系统 (Auto)。
   - 在“设置”页面添加主题切换选项。
   - 使用 CSS 变量或 Tailwind 类名进行主题适配。
   - 监听 uTools 的主题变化事件（如果存在）。
