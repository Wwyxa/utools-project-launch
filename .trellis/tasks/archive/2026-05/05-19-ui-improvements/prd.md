# UI 改进

## Goal

改进项目启动器的几个高频 UI 和交互问题：编辑项目时启动命令排序按钮反馈更明确；设置页默认终端的自定义命令输入框只在选择自定义终端时可编辑；运行日志长内容滚动更容易控制且不会把用户困在日志区域；项目列表卡片不因部分项目缺少说明而出现底部留白；项目列表支持用户调整展示顺序。

## What I Already Know

- 用户反馈项目卡片在有无项目说明混排时会为了对齐而在卡片下方出现空白。
- 用户希望项目列表顺序可修改，例如置顶、按某种顺序排序等。
- 当前编辑项目窗口的启动命令行已有上移/下移按钮，但悬浮和点击反馈较弱。
- 设置页默认终端中，自定义命令输入框当前始终可编辑。
- 运行日志组件是项目详情脚本页底部的内嵌滚动区域，长日志时滚动上下不便。
- 当前项目数据以数组形式持久化，项目顺序可通过重排数组保存，不必修改存储 schema。
- 上一版在每张项目卡片底部常驻置顶/上移/下移按钮，视觉占用过大，排序价值不明显。
- 上一版运行日志的回到顶部/底部按钮在实际窗口中无效，需要修复真实滚动目标。
- 运行日志和 Git 树都有内层滚动困住用户的问题，内容过长时不容易回到详情页上半部分。
- 运行日志和 Git 树不应使用固定黑色窗口，应跟随浅色/深色主题。

## Assumptions

- 项目列表顺序默认仍使用当前持久化数组顺序；新增项目继续出现在列表前面。
- 项目排序交互采用“置顶 + 上移/下移”，不做本次拖拽或规则排序。
- 自定义终端命令的已填写内容在切换到其他终端选项时保留，但输入框禁用，避免误改。
- 日志滚动改进优先做可见控制和滚动链路优化，不引入新的独立日志弹窗。
- 卡片布局应保持紧凑，不为了网格对齐强制撑高单个卡片内容。
- 项目排序控件应降低常驻视觉噪音，优先采用“排序模式”或更轻的入口，而不是挤占每张卡片底部主操作区。

## Requirements

- 编辑项目窗口的启动命令上移/下移按钮应有清晰 hover、pressed、focus、disabled 状态。
- 设置页默认终端中，“自定义命令”输入框只在选中“自定义命令”后可用，未选中时视觉上禁用且不能输入。
- 运行日志区域应支持便捷回到底部、回到顶部，并减少日志内层滚动阻断页面滚动的问题。
- 日志较长时新增输出不应强行打断用户正在查看历史日志；只有用户在底部附近时才自动跟随到底部。
- 项目卡片缺少说明时不应保留说明占位导致卡片底部出现不自然空白。
- 项目列表应支持通过置顶、上移、下移修改顺序，且修改后的顺序持久保存。
- 项目列表排序 UI 应比每张卡片常驻三枚排序按钮更清爽，普通浏览状态下不影响打开终端/目录/编辑/删除等高频操作。
- 运行日志回到顶部/底部按钮必须作用于真实滚动容器。
- Git 树窗口应获得与运行日志一致的长内容滚动体验：可快速回到顶部/底部，滚到边界后能继续滚动外层详情页。
- 运行日志和 Git 树的背景、文本和状态色应使用主题 token，跟随浅色/深色模式切换。

## Acceptance Criteria

- [ ] 编辑项目窗口中启动命令排序按钮 hover/active/focus/disabled 状态可明显区分。
- [ ] 设置页未选择自定义终端时，自定义命令输入框 disabled，并显示禁用态；选择自定义命令后恢复可编辑。
- [ ] 长日志可以用按钮快速回到顶部/底部，滚轮滚到日志边界时能继续滚动外层详情页面。
- [ ] 用户查看历史日志时新增日志不会自动把视图拉到底部；在底部附近时仍自动跟随。
- [ ] 项目列表卡片在部分项目没有说明时不出现额外留白，整体更紧凑。
- [ ] 项目卡片提供置顶、上移、下移排序操作，排序结果在刷新后保留。
- [ ] 项目排序控件在普通浏览时不常驻挤占卡片主操作区，进入排序模式或轻量入口后再显示。
- [ ] 运行日志顶部/底部按钮在实际窗口中可用。
- [ ] Git 树和运行日志的长内容滚动都能在边界自然转交外层详情页滚动。
- [ ] 运行日志和 Git 树在浅色/深色主题下都使用对应主题颜色，不再固定黑底。
- [ ] `npm run lint` 通过。
- [ ] `npm run build` 通过。

## Definition of Done

- 变更符合前端现有 Vue + Pinia + Tailwind 风格。
- 不引入不必要的新依赖。
- 不破坏现有项目导入/导出和旧配置读取。
- 更新必要的 i18n 文案。

## Out of Scope

- 不实现拖拽排序。
- 不实现多维复杂排序规则配置（如按状态、时间、类型自动排序并持久切换）。
- 不重写终端日志为完整终端模拟器。
