## Goal

消除首页（Dashboard）首屏"工具面板"观感，让它一打开就具备应用感与信息锚点。三处改造：顶部增加概览统计带、重做空状态、优化项目卡片信息层次。

## Requirements

### 1. 概览统计带（Overview Strip）

- 在 Dashboard 工具栏（搜索 + 操作按钮）下方、分组筛选 chip 与卡片网格之间，新增一条紧凑的概览统计带。
- 展示**即时可得、不依赖额外 Git 刷新**的指标（避免首页展示不可靠数据）：
  - 项目总数（可用 + 不可用）
  - 运行中（存在 `RUNNING`/`STOPPING` 脚本的项目数）
  - 异常/不可用（`status === ERROR` 或 `pathExists === false` 的项目数）
  - 最近活动（所有项目中最近一次 `lastUpdated`/`gitLatestCommitAt` 的相对时间）
- 每个指标：图标 + 数值 + 标签；运行中用绿色强调，异常用红/橙，其余中性。
- 响应式：窄窗口下指标卡自适应换行或收缩，不破坏工具栏布局。
- 项目数为 0 时，概览带不显示（让位于空状态）。

### 2. 空状态重做

- 将现有"虚线框 + 一行字 + 一个按钮"升级为有视觉层次的引导状态：
  - 一个较大的主题图标（如 `PackageOpen` / `FolderPlus`）。
  - 标题文案（如"还没有项目"）。
  - 副标题/引导文案（说明可做什么）。
  - 主操作：添加项目（保持现有 `openCreateProjectForm`）。
  - 次操作：从配置导入（复用 `store.importProjectConfig`），降低首次使用门槛。
- 保留与现有空状态相同的触发条件（`store.visibleProjects.length === 0`）。

### 3. 项目卡片信息层次优化

- 在不改变卡片网格尺寸（`minmax(15.5rem,1fr)`）与已有交互的前提下，优化 [ProjectCard.vue](src/components/project/ProjectCard.vue) 的视觉层次：
  - 统一并适度上调次要信息（描述/路径/时间）字号下限，减少 `text-[10px]` 的使用，提升可读性。
  - 增加卡片内关键区域间的呼吸感（脚本按钮区与底部操作区之间）。
  - 路径信息进一步弱化（降低视觉权重），让名称与运行状态成为视觉焦点。
  - 保持左侧状态色条 + 运行中徽章 + 悬停操作浮现等既有机制不变。
- 不新增卡片信息字段，仅调整呈现层次。

## Constraints

- 不引入新依赖。
- 不改 store 持久化结构、不改 bridge/preload 接口；概览带指标全部基于现有 store getters/state 派生（可在组件内用 `computed` 聚合，或新增纯派生 getter）。
- 双主题成立，只用语义 token。
- 新增可见文案在 `zh-CN` / `en-US` 两 locale 的 `dashboard` 块中对齐 key。
- 不破坏现有交互：搜索、分组筛选、排序拖拽、刷新、卡片选择/编辑/删除、脚本启停。
